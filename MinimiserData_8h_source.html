8 // Description
9 //
10 // Base class for handling data produced by the SiliconEnergyMinimiser. The
11 // minimiser function usually just returns the A value it has determined to be
12 // the most likely for a given Z (i.e. It returns an integer). Therefore to
13 // obtain more interesting information about how well the algorithm is
14 // performing, we need to pass in an (optional) pointer to a data class such as
15 // this one for the minimiser implementation to populate.
16 //
17 // This class handles only the essential data produced by any hypothetical
18 // minimiser implementation. For data specific to the implementation you must
19 // create an inheriting class (See MonoMinimiserData and ThreadedMinimiserData).
20 //
21 // Peter C. Wigg <peter.wigg.314159@gmail.com>
