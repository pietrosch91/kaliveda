4 //BinaryX_t with X=8, 16, 32 or 64
5 //
6 //Classes representing binary numbers with 8, 16, 32 or 64 bits.
7 //
8 //The following are valid initialisations:
9 //
10 // BinaryX_t a,b,c; Hexa_t d;
11 // a = 26;
12 // b = "110001";
13 // c = d;
14 // BinaryX_t e("1111");
15 //
16 //Use Value() to obtain the decimal value of the number.
17 //Use String() to obtain a string containing the binary representation.
18 //Default behaviour is to use the smallest number of bits necessary (single '0' for zero).
19 //
20 //If you use SetNBits to set the number of bits, any value, even 0, will always be represented using that number of bits.
21 //Use SetNBits(0) to reset to default behaviour.
22 //You can also impose the number of bits by calling String(nbits).
23 //
24 //NOTA BENE: the number of bits affects only the representation of the number as
25 //a string of '1's and '0's.
26 //It does not in any way affect either the current value of the number or its maximum possible value (given by Max()).
27 //
28 //BIT NUMBERING CONVENTION
29 //It is the same as the ROOT macro BIT(i) i.e.
30 //
31 // Number = 01101110
32 // Bit = 76543210
33 //
34 //SetBit(UChar_t nbit, T val)
35 //---------------------------
36 //Set bit 'nbit' to 1 (the least significant bit is numbered 0).
37 //If "val" is given, bit 'nbit' is set to the same value as the equivalent bit in 'val'
38 //
39 //ResetBit(UChar_t nbit)
40 //----------------------
41 //Set bit 'nbit' to 0 (the least significant bit is numbered 0).
42 //
43 //TestBit(UChar_t nbit)
44 //---------------------
45 //Returns kTRUE if bit 'nbit' is equal to '1' (the least significant bit is numbered 0).
46 //
47 //Hexa()
48 //------
49 //Returns number in its hexadecimal form (see Hexa_t)
50 //
51 //WriteSubvalue(const T val, UChar_t msb, UChar_t nbits)
52 //------------------------------------------------------
53 //Write the value 'val' in the 'nbits' bits from bit 'msb' (Most Significant Bit) to bit (msb-nbits+1).
54 //
55 //i.e. suppose b is an 8-bit binary number and we want to write the decimal value 10 in the 4 bits from bit 2 to bit 5
56 //(remember that the bits are numbered from right to left, LSB=bit 0).
57 //If b="00000000" and we do b.WriteSubvalue(10, 5, 4), we will have b="00101000"
58 //
59 //Error if the following conditions are not respected:
60 //1 <= nbits <= fNBits 0 <= msb <= (fNBits-1) (msb-nbits+1)>=0
61 //
62 //Subvalue(UChar_t firstbit, UChar_t nbits)
63 //-----------------------------------------
64 //Returns the decimal value of a part of this binary number.
65 //('firstbit' is equivalent to 'msb' argument in WriteSubvalue).
66 //The bits (firstbit, firstbit-1, firstbit-2, ... , firstbit-nbits+1) [see BIT NUMBERING CONVENTION]
67 //are used.
68 //If firstbit-nbits+1 < 0 (i.e. there are not enough bits in the number)
69 //we use all the bits from firstbit down to bit 0.
70 //
71 //Max()
72 //-----
73 //Returns maximum value that can be stored in number, i.e. when all bits = 1
74 //
75 //Print()
76 //-------
77 //Print following info: string representation of binary number, number of bits, optional number of bits used for representation
78 //(default value 0), decimal value of number.
79 //
