◆ GetRotation()

void KVTenseur3::GetRotation ( TRotation &  rot)
virtual

Sets rotation matrix corresponding to eigenvectors of diagonalized tensor. According to ROOT physics vectors convention, this is an 'active' rotation (see TRotation class description). Note that KVParticle::SetFrame automatically uses this matrix in such a way as to describe particles in the coordinate frame corresponding to the rotated axes.

By convention, the three eigenvectors (V1, V2, V3) are numbered in increasing order of size of the corresponding eigenvalue (V3 <–> largest eignevalue <–> "flow" axis). The new coordinate axes are defined (arbitrarily) as follows : V1 —> X' V2 —> Y' V3 —> Z' make orthonormal basis vectors from 3 eignevectors
