12 <h2>KVHashList</h2>
13 <h4>Extended version of ROOT THashList</h4>
14 This collection class uses a THashList for quick look-up of objects based on
15 the TString::Hash() value of their name, and adds all the extra functionality
16 defined in KVSeqCollection. Automatic rehashing of the list is enabled by default
17 (with rehash level = 2), unlike THashList (disabled by default).
18 <h3>Sorting the list</h3>
19 As THashList::Sort() is defined, we implemented a KVHashList::Sort() method which takes exactly
20 the same argument as THashList::Sort().

58 KVHashList::KVHashList(Int_t capacity, Int_t rehash)
59  : KVSeqCollection()
60 {
61  // Create a THashList object. Capacity is the initial hashtable capacity
62  // (i.e. number of slots), by default kInitHashTableCapacity = 17, and
63  // rehash is the value at which a rehash will be triggered. I.e. when the
64  // average size of the linked lists at a slot becomes longer than rehash
65  // then the hashtable will be resized and refilled to reduce the collision
66  // rate to about 1. The higher the collision rate, i.e. the longer the
67  // linked lists, the longer lookup will take.
68  // The default value of rehash = 2 (minimum allowed value) - automatic
69  // rehashing is enabled by default. If rehash=0 the table will
70  // NOT automatically be rehashed. Use Rehash() for manual rehashing.
71  // WARNING !!!
72  // If the name of an object in the HashList is modified, The hashlist
73  // must be Rehashed
74 
75  fCollection = new THashList(capacity, rehash);
76  fCollection->SetName(GetName());
77 }

123 void KVHashList::Rehash(Int_t newCapacity)
124 {
125  // Rehash the hashlist.
126  // This needs to be done in two cases:
127  // 1) If the collision rate becomes too high (i.e. the average size of the
128  // linked lists become too long - use AverageCollisions() to check if you
129  // need to rehash.) then lookup efficiency decreases since relatively long
130  // lists have to be searched every time.
131  // To improve performance rehash the hashtable, increasing
132  // the number of slots. This method resizes the table to newCapacity slots
133  // and refills the table.
134  // 2) If the names of any objects in the list CHANGE, you must
135  // rehash the list (as the lookup of objects depends on the TString::Hash()
136  // value calculated from their name when they are added to the list).
137  // In this case it is not necessary to increase the capacity of the list,
138  // just call Rehash() with no arguments.
139 
140  if (!newCapacity) newCapacity = fCollection->GetSize();
141  ((THashList*)fCollection)->Rehash(newCapacity);
142 }
