24 <h2>KVBatchSystem</h2>
25 <h4>Base class for interface to a batch job management system</h4>
26 
27 <p>The different "batch" systems available are described in the $KVROOT/KVFiles/.kvrootrc
28 or the user's $HOME/.kvrootrc, as in the following example:</p>
29 <pre>
30 BatchSystem: Xterm
31 Xterm.BatchSystem.Title: Execute task in an X-terminal window
32 Xterm.BatchSystem.DefaultJobOptions: -T #JobName# -e
33 Xterm.BatchSystem.JobSubCmd: xterm
34 Xterm.BatchSystem.JobScript: "$KVROOT/bin/KaliVedaAnalysis -b -n"&
35 </pre>
36 <p>For each named batch system, there may be a corresponding plugin which
37 defines the actual class to use, if it is necessary to extend the functionality of the
38 KVBatchSystem base class, as for example:</p>
39 <pre>
40 Plugin.KVBatchSystem: Xterm KVRootBatch KVMultiDet "KVRootBatch(const Char_t*)"
41 </pre>
42 <p>The "default" batch system is defined as follows:</p>
43 <pre>
44 #Default batch system
45 Default.BatchSystem: Xterm
46 </pre>
47 <p>It can be accessed via the batch system manager (see <a href="KVBatchSystemManager.html">KVBatchSystemManager</a>):</p>
48 <pre>
49 ...
50 #include "KVBatchSystemManager.h"
51 ...
52 KVBatchSystem* default = gBatchSystemManager->GetDefaultBatchSystem();
53 </pre>
54 <h4>Access to batch parameters in user analysis class</h4>
55 <p>Global pointer gBatchSystem is 0 if analysis task is not run in batch mode
56 If it is non-zero, it gives access to the batch system used to run the task. Therefore, if the user
57 puts</p><pre>
58 #include "KVBatchSystem.h"
59 </pre>
60 <p>in her analysis code, she can then use gBatchSystem as follows:</p>
61 <pre>
62 if( gBatchSystem ) {
63  //running in batch. name results file after job name.
64  new TFile( Form( "%s.root", gBatchSystem->GetJobName() ), "recreate");
65 }
66 else {
67  //not running in batch. fixed name.
68  new TFile("Results.root", "recreate");
69 }
70 </pre>
71 <!-- */

164  //script for batch job
165  SetJobScript(gEnv->GetValue(Form("%s.BatchSystem.JobScript", name), ""));
166  //set default job options

207 const Char_t* KVBatchSystem::GetJobSubCmdLine()
208 {
209  //Builds and returns static string containing full command line necessary to submit a batch
210  //job. This is constructed as follows:
211  //
212  // [JobSubCmd] [default options] [par1 val1] [par2 val2] ... [JobScript]
213  //
214  //The 'default options' can be set with SetDefaultJobOptions().
215  //
216  //The 'par1,val1' pairs are the named parameters held in the fParList KVParameterList.
217  //For each parameter in the list, the name of the parameter is used for 'par1' and the value
218  //of the parameter is used for 'val1' etc. etc.
219  //e.g. if the job submission command requires an option to be passed with the following
220  //syntax
221  // [JobSubCmd] -Noption
222  //one of the parameters in the list should be called "-N" and its value should be 'option'
223  //
224  //In order to add a simple flag such as
225  // [JobSubCmd] -V
226  //add a parameter called "-V" with value "".
227  //
228  //The special variable #JobName# can be used anywhere and will be replaced by the
229  //jobname returned by GetJobName() at the moment of job submission.
230 
231  static TString command_line;
232  command_line.Form("%s %s ", fJobSubCmd.Data(), fDefOpt.Data());
233  if (fParList.GetNpar()) {
234  for (int i = 0; i < fParList.GetNpar(); i++) {
235  KVNamedParameter* par = fParList.GetParameter(i);
236  command_line += par->GetName();
237  if (par->GetTString() != "") {
238  command_line += par->GetString();
239  }
240  command_line += " ";
241  }
242  }
243  command_line += fJobScript;
244  //replace #JobName# with name of current job
245  command_line.ReplaceAll("#JobName#", GetJobName());
246  return command_line.Data();
247 }

288 void KVBatchSystem::SubmitJob()
289 {
290  //Submits a job to batch system, i.e. executes the string formed by GetJobSubCmdLine,
291  //if all necessary parameters have been given (any missing ones will be asked for).
292  //Parameters specific to a given batch system can be added my modifying the
293  //CheckJobParameters() method for the associated child class.
294 
295  //set environment variables required by KaliVedaAnalysis batch executable
296  gSystem->Setenv("KVBATCHNAME", GetJobName());
297  gSystem->Setenv("KVLAUNCHDIR", gSystem->WorkingDirectory());
298  cout << GetJobSubCmdLine() << endl;
299  if (fAnalyser) {
300  fAnalyser->WriteBatchEnvFile(GetJobName());
301  gSystem->Setenv("KVANALYSER", fAnalyser->ClassName());
302  }
303  gSystem->Exec(GetJobSubCmdLine());
304 }

388 void KVBatchSystem::ChangeDefJobOpt(KVDataAnalyser* da)
389 {
390  // PRIVATE method called by SubmitTask() at moment of job submission.
391  // Depending on the current environment, the default job submission options
392  // may be changed by this method.
393  // For example, default options may be changed depending on the analysis
394  // task to be performed. We look for an environment variable of the form:
395  //
396  // [batch system name].BatchSystem.DefaultJobOptions.[analysis task name]
397  //
398  // and if found we use the options defined by it. If not, we use the default
399  // job options defined by
400  // [batch system name].BatchSystem.DefaultJobOptions
401 
402  TString tmp = gEnv->GetValue(Form("%s.BatchSystem.DefaultJobOptions.%s",
403  GetName(), da->GetAnalysisTask()->GetName()), "");
404  if (tmp.Length()) {
405  Info("ChangeDefJobOpt", "Changing default batch options for task %s.", da->GetAnalysisTask()->GetName());
406  Info("ChangeDefJobOpt", "Batch job options for this job are : %s", tmp.Data());
407  SetDefaultJobOptions(tmp.Data());
408  } else {
409  tmp = gEnv->GetValue(Form("%s.BatchSystem.DefaultJobOptions", GetName()), "");
410  SetDefaultJobOptions(tmp.Data());
411  }
412 }

425 const Char_t* KVBatchSystem::GetJobName() const
426 {
427  //Returns name of batch job, either during submission of batch jobs or when an analysis
428  //task is running in batch mode (access through gBatchSystem global pointer).
429  //
430  // Depending on the batch system, some sanitization of the jobname may be required
431  // e.g. to remove "illegal" characters from the jobname. This is done by SanitizeJobName()
432  // before the jobname is returned.
433 
434  if (!fAnalyser) {
435  //stand-alone batch submission ?
436  fCurrJobName = fJobName;
437  } else {
438  //replace any special symbols with their current values
439  fCurrJobName = fAnalyser->ExpandAutoBatchName(fJobName.Data());
440  }
441  SanitizeJobName();
442  return fCurrJobName.Data();
443 }
