20 <h2>KVEventSelector</h2>
21 <h4>General purpose analysis class for TTrees containing KVEvent objects</h4>
22  Use this TSelector to analyse data in TTrees containing a branch with KVEvent-derived objects:
23 
24 ~~~~~~~~~~~~~~~~
25  my_tree->Process("KVEventSelector", "[options]");
26 ~~~~~~~~~~~~~~~~
27 
28  where `"[options]"` is the list of options in the form `"BranchName=toto, ..."`.
29 
30  The following options MUST be given:
31 
32 ~~~~~~~~~~~~~~~~
33  BranchName: name of branch containing the events
34 ~~~~~~~~~~~~~~~~
35 
36  The following is an optional option:
37 
38 ~~~~~~~~~~~~~~~~
39  EventsReadInterval: print info on analysis every N events instead of default value
40 ~~~~~~~~~~~~~~~~
41 
42  Any other options can be defined by the user and parsed in her analysis class
43  with methods IsOptGiven() and GetOpt()
44 
45 ## Usage
46 ### Global Variables
47 Global variables for the analysis should be declared in InitAnalysis() using method AddGV().
48 These variables are calculated automatically for each event __before__ user's Analysis()
49 method is called.
50 
51  If the user needs to define new reference frames for the data which must exist before global variables
52  are automatically calculated in the analysis loop, she can do so by overriding the method
53  SetAnalysisFrame(), like so:
54 
55 ~~~~~~~~~~~~~~~~
56  MyAnalysis::SetAnalysisFrame()
57  {
58  static TVector3 cmvelocity(0,0,3.0);
59  GetEvent()->SetFrame("CM", cmvelocity);
60  }
61 ~~~~~~~~~~~~~~~~
62 
63 ### Particle selection for global variables
64 In order to control which particles are included in the calculation of global variables,
65 define the selection using KVParticleCondition and set it by calling SetParticleConditions()
66 in InitAnalysis() or InitRun().
67 
68 This selection also applies to any iteration over "OK" particles in Analysis().
69 
70 ### Use with PROOF(lite)
71  In order to use a KVEventSelector with PROOF:
72  - declare any histograms with method AddHisto(TH1*)
73  e.g. in InitAnalysis() do:
74 
75 
76 ~~~~~~~~~~~~~~~~
77  void MySelector::InitAnalysis()
78  {
79  AddHisto( new TH2F("toto", "tata", 100, 0, 0, 500, 0, 0) );
80  }
81 ~~~~~~~~~~~~~~~~
82 
83  Histograms can also be declared 'on the fly' in the Analysis() method in
84  the same way;
85 
86  - for TTrees, first call CreateTreeFile() with name of file for TTree(s)
87  (by default, histograms and TTrees are written in different files - but see below),
88  then declare all trees using method AddTree(TTree*)
89  e.g. in InitAnalysis() do:
90 
91 ~~~~~~~~~~~~~~~~
92  void MySelector::InitAnalysis()
93  {
94  CreateTreeFile("MyTrees.root");
95  TTree* aTree = new TTree("t1", "Some Tree");
96  aTree->Branch(...) etc.
97  AddTree(aTree);
98  }
99 ~~~~~~~~~~~~~~~~
100 
101  - if you want (not obligatory), you can use methods FillHisto() and
102  FillTree() in your Analysis() method;
103 
104  - to save histograms to file in EndAnalysis(), call method
105  SaveHistos() in EndAnalysis() with the name of the file to be created
106 
107  - the file declared with CreateTreeFile() will be automatically written
108  to disk at the end of the analysis.
109 
110 ### Histograms & TTrees in the same output file
111  If you want all results of your analysis to be written in a single file
112  containing both histos and trees, put the following in the list of options:
113 
114 ~~~~~~~~~~~~~~~~
115  CombinedOutputFile=myResults.root
116 ~~~~~~~~~~~~~~~~
117 
118  or call method SetCombinedOutputFile() with the required filename in your InitAnalysis() method;
119 
120  - do not call SaveHistos() in EndAnalysis(), and make sure you call CreateTreeFile() without giving a name (the
121  resulting intermediate file will have a default name allowing it to be found at the end of the analysis)
122 
123 ### Generating & saving profiles, divided histograms, etc.
124 If, at the end of processing, you want to generate a histogram from one or more histograms filled in your analysis,
125 for example generate a TProfile from a 2D histogram, or store the result of dividing one histogram by the other,
126 you need to do the following:
127 
128  - do the processing in the EndRun() method, not in EndAnalysis() - by the time EndAnalysis() is called it is too late,
129  the output file has been closed and written to disk
130  - add the resulting histogram to the output list like so:
131 
132 ~~~~~~~~~~
133  MyEventSelector::EndRun()
134  {
135  TProfile* my_profile = ((TH2*)GetHisto("my_histo_2D"))->ProfileX("name_of_my_profile");
136  GetOutputList()->Add(my_profile);
137  }
138 ~~~~~~~~~~
139 */

451 {
452  // The Process() function is called for each entry in the tree (or possibly
453  // keyed object in the case of PROOF) to be processed. The entry argument
454  // specifies which entry in the currently loaded tree is to be processed.
455  // It can be passed to either KVEventSelector::GetEntry() or TBranch::GetEntry()
456  // to read either all or the required parts of the data. When processing
457  // keyed objects with PROOF, the object is already loaded and is available
458  // via the fObject pointer.
459  //
460  // This function should contain the "body" of the analysis. It can contain
461  // simple or elaborate selection criteria, run algorithms on the data
462  // of the event and typically fill histograms.
463  //
464  // Processing will abort cleanly if static flag fCleanAbort has been set
465  // by some external controlling process.
466  //
467  // Use fStatus to set the return value of TTree::Process().
468  //
469  // The return value is currently not used.
470 
471  if (gDataAnalyser && gDataAnalyser->AbortProcessingLoop()) {
472  // abort requested by external process
473  Abort(Form("Job received KILL signal from batch system after %lld events - batch job probably needs more CPU time (see end of job statistics)", fEventsRead), kAbortFile);
474  return kFALSE;
475  }
476 
477  fTreeEntry = entry;
478 
479  if (!(fEventsRead % fEventsReadInterval) && fEventsRead) {
480  Info("Process", " +++ %lld events processed +++ ", fEventsRead);
481  ProcInfo_t pid;
482  if (gSystem->GetProcInfo(&pid) == 0) {
483  cout << " ------------- Process infos -------------" << endl;
484  printf(" CpuSys = %f s. CpuUser = %f s. ResMem = %f MB VirtMem = %f MB\n",
485  pid.fCpuSys, pid.fCpuUser, pid.fMemResident / 1024., pid.fMemVirtual / 1024.);
486  }
487  }
488  GetEntry(entry);
489  if (gDataAnalyser) gDataAnalyser->preAnalysis();
490  fEventsRead++;
491  if (GetEvent()) {
492  KVNucleus* part = 0;
493  //apply particle selection criteria
494  if (fPartCond) {
495  part = 0;
496  GetEvent()->ResetGetNextParticle();

503

672 KVVarGlob* KVEventSelector::AddGV(const Char_t* class_name,
673  const Char_t* name)
674 {
675  //Add a global variable to the list of variables for the analysis.
676  //
677  //`"class_name"` must be the name of a valid class inheriting from KVVarGlob, e.g. any of the default global
678  //variable classes defined as part of the standard KaliVeda package (see the [global variables module](group__Globvars.html)),
679  //or the name of a user-defined class (see below).
680  //
681  //`"name"` is a unique name for the new global variable object which will be created and added to the internal
682  //list of global variables. This name can be used to retrieve the object (see GetGV()) in the user's analysis.
683  //
684  //Returns pointer to new global variable object in case more than the usual default initialisation is necessary.
685  //
686  //### User-defined global variables
687  //The user may use her own global variables in an analysis class, without having to add them to the main libraries.
688  //If the given class name is not known, it is assumed to be a user-defined class and we attempt to compile and load
689  //the class from the user's source code. For this to work, the user must:
690  //
691  // 1. add to the ROOT macro path the directory where her class's source code is kept, e.g. in `$HOME/.rootrc` add the following line:
692  //
693  //~~~~~~~~~~~~~~~
694  // +Unix.*.Root.MacroPath: $(HOME)/myVarGlobs
695  //~~~~~~~~~~~~~~~
696  //
697  // 2. for each user-defined class, add a line to $HOME/.kvrootrc to define a "plugin". E.g. for a class called MyNewVarGlob,
698  //
699  //~~~~~~~~~~~~~~~
700  // +Plugin.KVVarGlob: MyNewVarGlob MyNewVarGlob MyNewVarGlob.cpp+ "MyNewVarGlob()"
701  //~~~~~~~~~~~~~~~
702  //
703  // It is assumed that `MyNewVarGlob.h` and `MyNewVarGlob.cpp` will be found in `$HOME/myVarGlobs` (in this example).
704 
705  KVVarGlob* vg = 0;
706  TClass* clas = gROOT->GetClass(class_name);
707  if (!clas) {
708  //class not in dictionary - user-defined class ? Look for plugin.
709  TPluginHandler* ph = KVBase::LoadPlugin("KVVarGlob", class_name);
710  if (!ph) {
711  //not found
712  Error("AddGV(const Char_t*,const Char_t*)",
713  "Called with class_name=%s.\nClass is unknown: not in standard libraries, and plugin (user-defined class) not found",
714  class_name);
715  return 0;
716  }
717  else {
718  vg = (KVVarGlob*) ph->ExecPlugin(0);
719  }
720  }
721  else if (!clas->InheritsFrom("KVVarGlob")) {
722  Error("AddGV(const Char_t*,const Char_t*)",
723  "%s is not a valid class deriving from KVVarGlob.",
724  class_name);
725  return 0;
726  }
727  else {
728  vg = (KVVarGlob*) clas->New();
729  }
730  vg->SetName(name);
731  AddGV(vg);
732  return vg;
733 }

779 void KVEventSelector::SetParticleConditions(const KVParticleCondition& cond, const KVString& upcast_class)
780 {
781  //Use this method to set criteria for selecting particles to include in analysis.
782  //The criteria defined in the KVParticleCondition object will be applied to every
783  //particle and if they are not satisfied the particle's
784  //"OK" flag will be set to false, i.e. the particle's IsOK() method will return kFALSE,
785  //and the particle will not be included in iterations such as GetEvent()->GetNextParticle("OK").
786  //Neither will the particle be included in the evaluation of any global variables.

788  //This method must be called in the user's InitAnalysis() or InitRun() method.

790  //If the methods used in the condition are not defined for KVNucleus, you can give the

792  //hand (SetParticleConditionsParticleClassName)
793 
794  if (!fPartCond) fPartCond = new KVParticleCondition(cond);
795  else *fPartCond = cond;
796  if (upcast_class != "") fPartCond->SetParticleClassName(upcast_class);
797  else if (fPartName) fPartCond->SetParticleClassName(fPartName);
