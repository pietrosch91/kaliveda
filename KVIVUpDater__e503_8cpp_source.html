81  // IMPORTANT!
82  //
83  // An exception can be thrown here (intentionally, as one can't pass a
84  // return code back up the stack). The reason for this is that the function
85  // signature must match that of KVINDRAUpDater::SetParameters (which has a
86  // void return type).
87  //
88  // The advantage of throwing an exception is that the error can be handled
89  // properly further up the stack in KVIVSelector::Notify.
90  // KVIVSelector::Notify will automatically end the current run if it
91  // catches this exception.
92  //
93  // The alternative is the unexpected termination of the parent process,
