21 <h2>KVDataAnalysisTask</h2>
22 <h4>Defines a data analysis task</h4>
23 <p>
24 Data analysis tasks are defined in the files $KVROOT/KVFiles/.kvrootrc and the user's
25 $HOME/.kvrootrc, as in the following examples:
26 </p>
27 <pre>
28 DataAnalysisTask: Reconstruction
29 Reconstruction.DataAnalysisTask.Title: Event reconstruction from raw data (raw->recon)
30 Reconstruction.DataAnalysisTask.Prereq: raw
31 Reconstruction.DataAnalysisTask.Analyser: INDRARawDataRecon
32 </pre>
33 <p>
34 Each task has a name ("Reconstruction"), a descriptive title ("Event reconstruction from raw data (raw->recon)"),
35 a prerequisite type of data on which the analysis is performed ("raw"), and the name of a
36 <a href="KVDataAnalyser.html">KVDataAnalyser</a> plugin which is used to perform the analysis ("INDRARawDataRecon").
37 For a given <a href="KVDataSet.html">dataset</a>, only analysis tasks whose prerequisite data type is available can be
38 performed.
39 </p>
40 <pre>
41 +DataAnalysisTask: ReconAnalysis
42 ReconAnalysis.DataAnalysisTask.Title: Analysis of reconstructed events (recon)
43 ReconAnalysis.DataAnalysisTask.Prereq: recon
44 ReconAnalysis.DataAnalysisTask.Analyser: INDRAReconData
45 ReconAnalysis.DataAnalysisTask.UserClass: yes
46 ReconAnalysis.DataAnalysisTask.UserClass.Base: INDRASelector/TSelector
47 +Plugin.TSelector: INDRASelector KVSelector KVIndra "KVSelector()"
48 </pre>
49 <p>
50 This analysis task requires the user to supply an analysis class ("...UserClass: yes"); the class must
51 be derived from a plugin base class, <a href="KVSelector.html">KVSelector</a>, which is not defined in
52 the basic KaliVeda library (see <a href="#KVDataAnalysisTask:SetUserBaseClass">SetUserBaseClass()</a> for details).
53 It will be automatically loaded when required.
54 </p>
55 <pre>
56 +DataAnalysisTask: RawAnalysis
57 RawAnalysis.DataAnalysisTask.Title: Analysis of raw data
58 RawAnalysis.DataAnalysisTask.Prereq: raw
59 RawAnalysis.DataAnalysisTask.Analyser: UserClass
60 RawAnalysis.DataAnalysisTask.UserClass: yes
61 RawAnalysis.DataAnalysisTask.UserClass.Base: INDRARawDataAnalyser
62 +Plugin.KVDataAnalyser: INDRARawDataAnalyser KVINDRARawDataAnalyser KVIndra "KVINDRARawDataAnalyser()"
63 </pre>
64 <p>
65 This analysis task also requires the user to supply an analysis class ("...UserClass: yes"), but as the base class
66 for the user's class is itself derived from KVDataAnalyser (via a plugin: see <a href="#KVDataAnalysisTask:SetUserBaseClass">SetUserBaseClass()</a> for details)
67 it is the user's class which will be used to perform the analysis ("...Analyser: UserClass").
68 </p>
69 <pre>
70 +DataAnalysisTask: ReconIdent
71 ReconIdent.DataAnalysisTask.Title: Identification of reconstructed events (recon->ident)
72 ReconIdent.DataAnalysisTask.Prereq: recon
73 ReconIdent.DataAnalysisTask.Analyser: INDRAReconData
74 ReconIdent.DataAnalysisTask.UserClass: no
75 ReconIdent.DataAnalysisTask.UserClass.Base: INDRAReconIdent/TSelector
76 +Plugin.TSelector: INDRAReconIdent KVReconIdent KVIndra "KVReconIdent()"
77 </pre>
78 <p>
79 This analysis task does not require a user-supplied analysis class ("...UserClass: no") as it supplies a default
80 analysis class to be used by the analyser (via a plugin: <a href="KVReconIdent.html">KVReconIdent</a>.
81 See <a href="#KVDataAnalysisTask:SetUserBaseClass">SetUserBaseClass()</a> for details).
82 </p>
83 <h4>Dataset-dependent analysis tasks</h4>
84 <p>
85 Certain characteristics (analyser plugin, base class for user analysis class)
86 of analysis tasks can be "tuned" for different datasets. For example, to change the
87 KVDataAnalyser plugin to use for the "Reconstruction" task for the dataset "INDRA_e494s":
88 </p>
89 <pre>
90 INDRA_e494s.Reconstruction.DataAnalysisTask.Analyser: IVRawDataRecon_e494s
91 </pre>
92 <p>
93 where the plugin is defined as:
94 </p>
95 <pre>
96 +Plugin.KVDataAnalyser: IVRawDataRecon_e494s KVIVRawDataReconstructor VAMOS "KVIVRawDataReconstructor()"
97 </pre>
98 <!-- */

297 void KVDataAnalysisTask::SetUserBaseClass(const Char_t* base_name)
298 {
299  // Set the class from which user's analysis class must derive.
300  // This is normally read in from the following environment variable:
301  //
302  // TaskName.DataAnalysisTask.UserClass.Base: [base_name]
303  //
304  // If base_name is not the name of a known (i.e. currently contained in the ROOT
305  // dictionary class table) class, it is assumed to be the uri of a plugin class.
306  // By default, we assume that the plugin extends the KVDataAnalyser class, but
307  // this can be changed using the following syntax:
308  //
309  // TaskName.DataAnalysisTask.UserClass.Base: [uri]/[base class]
310  //
311  // Example 1:
312  // The user analysis class for task "RawAnalysis" must derive from class
313  // KVIVRawDataAnalyser which is defined in library libVAMOS.so (not loaded by default):
314  //
315  // RawAnalysis.DataAnalysisTask.UserClass.Base: IVRawDataAnalyser
316  // +Plugin.KVDataAnalyser: IVRawDataAnalyser KVIVRawDataAnalyser VAMOS "KVIVRawDataAnalyser()"
317  //
318  // Example 2:
319  // The user analysis class for task "ReconIdent" must derive from class
320  // KVIVReconIdent (itself derived from TSelector) which is defined in library libVAMOS.so (not loaded by default):
321  //
322  // ReconIdent.DataAnalysisTask.UserClass.Base: IVReconIdent/TSelector
323  // +Plugin.TSelector: IVReconIdent KVIVReconIdent VAMOS "KVIVReconIdent()"
324 
325  fBaseClass = base_name;
326  if (fBaseClass != "" && !TClass::GetClass(base_name)) {
327  // Base class is not in dictionary. Assume that base_name is a plugin.
328  fBaseIsPlugin = kTRUE;
329  KVString _base(base_name);
330  TObjArray* toks = _base.Tokenize("/");
331  if (toks->GetEntries() > 1) {
332  // base_name = uri/base
333  fPluginURI = ((TObjString*)toks->At(0))->String();
334  fPluginBase = ((TObjString*)toks->At(1))->String();
335  } else {
336  // base_name = uri. default plugin base class is KVDataAnalyser
337  fPluginURI = base_name;
338  fPluginBase = "KVDataAnalyser";
339  }
340  delete toks;
341  // check plugin exists
342  TPluginHandler* ph = gPluginMgr->FindHandler(fPluginBase.Data(), fPluginURI.Data());
343  if (!ph) {
344  Error("SetUserBaseClass", "Analysis task=%s :\nNo plugin found with URI=%s for base class %s",
345  GetName(), fPluginURI.Data(), fPluginBase.Data());
346  return;
347  }
348  // get name of plugin base class that user's analysis must derive from
349  fBaseClass = ph->GetClass();
350  if (gDebug > 1) Info("SetUserBaseClass", "Analysis task=%s :\nUser analysis class derives from %s, extension of %s in plugin %s",
351  GetName(), fBaseClass.Data(), fPluginBase.Data(), fPluginURI.Data());
352  }
353 }

364 Bool_t KVDataAnalysisTask::CheckUserBaseClassIsLoaded()
365 {
366  // This method checks that the base class for the user's analysis class is loaded.
367  // If this base class requires a plugin library to be loaded, it will be loaded.
368  // First we add any required extra ACliC include paths (taken from DataAnalysisTask.UserClass.ExtraACliCIncludes)
369  // If all is OK, returns kTRUE.
370  // Returns kFALSE if plugin cannot be loaded or class is simply unknown
371 
372  if (fExtraAClicIncludes != "") {
373  gSystem->AddIncludePath(fExtraAClicIncludes.Data());
374  Info("CheckUserBaseClassIsLoaded", "Added ACliC include path: %s", fExtraAClicIncludes.Data());
375  }
376  TClass* cl = TClass::GetClass(fBaseClass.Data()); // class in dictionary already ?
377  if (cl) return kTRUE;
378  if (fBaseIsPlugin) {
379  // base class is in a plugin library. load it now.
380  TPluginHandler* ph = LoadPlugin(fPluginBase.Data(), fPluginURI.Data());
381  if (ph) return kTRUE;
382  Error("CheckUserBaseClassIsLoaded", "Analysis task=%s : Failed to load plugin class=%s base=%s uri=%s",
383  GetName(), fBaseClass.Data(), fPluginBase.Data(), fPluginURI.Data());
384  } else {
385  Error("CheckUserBaseClassIsLoaded", "Analysis task=%s : User analysis base class %s is unknown",
386  GetName(), fBaseClass.Data());
387  }
388  return kFALSE;
389 }
