20 <h2>KVGANILDataReader</h2>
21 <h4>Reads GANIL acquisition files</h4>
22  Open and initialise a GANIL data file for reading.
23  If file cannot be opened, this object will be made a zombie. Do not use.
24  To test if file is open, use IsOpen().
25  The basename of the file (excluding any path) can be obtained using GetName()
26  The full pathname of the file can be obtained using GetTitle()
27  <br><br>
28  If the dataset corresponding to the data to be read is known i.e. if gDataSet has been defined and points
29  to the correct dataset, this will allow to build the necessary multidetector object if it has not already
30  been done, and to set the calibration parameters etc. as a function of the run number.
31  <br><br>
32  If not (i.e. if no information is available on detectors, calibrations, geometry, etc.),
33  then a list of KVACQParam objects will be generated and connected ready for reading the data.
34  <br><br>
35  To fill a TTree with all data in the file, do the following:<br>
36 <pre>
37 KVGANILDataReader* runfile = new KVGANILDataReader("run1.dat");
38 TFile* file = new TFile("run1.root","recreate");
39 TTree* T = new TTree("Run1", "Raw data for Run1");
40 runfile->SetUserTree(T);
41 while( runfile->GetNextEvent() ) ;
42 file->Write();
43 file->Close();
44 </pre><br>
45 See method <a href="#KVGANILDataReader:SetUserTree">SetUserTree()</a> for more details.
46 See below if you want to include a TTree containing scaler data in the file.
47 <h4>Scaler buffers management</h4>
48 By default, scaler buffers are ignored (<a href="GTGanilData.html#GTGanilData:SetScalerBuffersManagement">GTGanilData::SetScalerBuffersManagement</a>(GTGanilData::kSkipScaler)).
49 This can be changed by changing the value of<br>
50 <pre>
51 KVGANILDataReader.ScalerBuffersManagement: kSkipScaler
52 </pre>
53 <br>
54 For possible values, see <a href="GTGanilData.html#GTGanilData:SetScalerBuffersManagement">GTGanilData::SetScalerBuffersManagement</a>.<br><br>
55 You can also add a second TTree to the user tree generated above containing the values
56 of all scaler buffers written in the data file. The TTree will be called 'Scalers'.
57 You need to add "SCALERS" to the option given to method <a href="#KVGANILDataReader:SetUserTree">SetUserTree()</a> (see below).

256 void KVGANILDataReader::SetUserTree(TTree* T, Option_t* opt)
257 {
258  // To fill a TTree with the data in the current file, create a TTree:
259  // TFile* file = new TFile("run1.root","recreate");
260  // TTree* T = new TTree("Run1", "Raw data for Run1");
261  // and then call this method: SetUserTree(T)
262  // If you read all events of the file, the TTree will be automatically filled
263  // with data :
264  // while( runfile->GetNextEvent() ) ;
265  //
266  // Two different TTree structures are available, depending on the option string:
267  //
268  // opt = "arrays": [default]
269  //
270  // The TTree will have the following structure:
271  //
272  // *Br 0 :NbParFired : NbParFired/I = number of fired parameters in event
273  // *............................................................................*
274  // *Br 1 :ParNum : ParNum[NbParFired]/i = array of indices of fired parameters
275  // *............................................................................*
276  // *Br 2 :ParVal : ParVal[NbParFired]/s = array of values of fired parameters
277  //
278  // This structure is the fastest to fill and produces the smallest file sizes.
279  // In order to be able to directly access the parameters as if option "leaves" were used
280  // (i.e. one branch/leaf for each parameter), we add two aliases for each parameter to
281  // the tree:
282  // PARNAME = value of parameter if present in event
283  // PARNAME_M = number of times parameter appears in event
284  // Assuming that each parameter only appears at most once in each event, i.e. PARNAME_M=0 or 1,
285  // then
286  // root[0] T->Draw("PARNAME", "PARNAME_M")
287  // will histogram the value of PARNAME for each event in which it is present.
288  // (if the selection condition "PARNAME_M" is not used, the histogram will also be filled with a 0
289  // for each event in which PARNAME does not appear).
290  // N.B. the PARNAME alias is in fact the sum of the values of PARNAME in each event.
291  // If PARNAME_M>1 in some events, it is not the individual values but their sum which will
292  // be histogrammed in this case.
293  //
294  // Thus, if the data file has parameters called "PAR_1" and "PAR_2",
295  // the following command will work
296  //
297  // root[0] T->Draw("PAR_1:PAR_2", "PAR_1_M&&PAR_2_M", "col")
298  //
299  // even though no branches "PAR_1" or "PAR_2" exist.
300  //
301  //
302  //
303  // opt = "leaves":
304  //
305  // The TTree will have a branch/leaf for each parameter. This option is slower and produces
306  // larger files.
307  //
308  // If the option string contains both "arrays" and "leaves", then both structures will be used
309  // (in this case there is a high redundancy, as each parameter is stored twice).
310  //
311  // The full list of parameters is stored in a TObjArray in the list returned by TTree::GetUserInfo().
312  // Each parameter is represented by a TNamed object.
313  // In order to retrieve the name of the parameter with index 674 (e.g. taken from branch ParNum),
314  // do:
315  // TObjArray* parlist = (TObjArray*) T->GetUserInfo()->FindObject("ParameterList");
316  // cout << "Par 674 name = " << (*parlist)[674]->GetName() << endl;
317  //
318  //
319  // Automatic creation & filling of Scalers TTree
320  //
321  // give an option string containing "scalers", i.e. "leaves,scalers", or "ARRAYS+SCALERS", etc.
322  // a TTree with name 'Scalers' will be created, all scaler buffers will be written in it.
323 
324 
325  TString option = opt;
326  option.ToUpper();
327  make_arrays = option.Contains("ARRAYS");
328  make_leaves = option.Contains("LEAVES");
329  Bool_t make_scalers = option.Contains("SCALERS");
330  if (make_scalers) {
331  fGanilData->SetScalerBuffersManagement(GTGanilData::kAutoWriteScaler);
332  }
333 
334  fUserTree = T;
335  if (make_arrays) {
336  Int_t maxParFired = GetRawDataParameters()->GetEntries();
337  ParVal = new UShort_t[maxParFired];
338  ParNum = new UInt_t[maxParFired];
339  fUserTree->Branch("NbParFired", &NbParFired, "NbParFired/I");
340  fUserTree->Branch("ParNum", ParNum, "ParNum[NbParFired]/i");
341  fUserTree->Branch("ParVal", ParVal, "ParVal[NbParFired]/s");
342  }
343  if (make_leaves) {
344  TIter next_rawpar(GetRawDataParameters());
345  KVACQParam* acqpar;
346  while ((acqpar = (KVACQParam*)next_rawpar())) {
347  TString leaf;
348  leaf.Form("%s/S", acqpar->GetName());
349  // for parameters with <=8 bits only use 1 byte for storage
350  if (acqpar->GetNbBits() <= 8) leaf += "1";
351  fUserTree->Branch(acqpar->GetName(), *(acqpar->ConnectData()), leaf.Data());
352  }
353  }
354 
355 #if ROOT_VERSION_CODE > ROOT_VERSION(5,25,4)
356 #if ROOT_VERSION_CODE < ROOT_VERSION(5,26,1)
357  // The TTree::OptimizeBaskets mechanism is disabled, as for ROOT versions < 5.26/00b
358  // this lead to a memory leak
359  fUserTree->SetAutoFlush(0);
360 #endif
361 #endif
362 
363  // add list of parameter names in fUserTree->GetUserInfos()
364  // and if option="arrays" add aliases for each parameter & its multiplicity
365 
366  // TObjArray has to be as big as the largest parameter number in the list
367  // of raw data parameters. So first loop over parameters to find max param number.
368  UInt_t maxpar = 0;
369  TIter next(GetRawDataParameters());
370  KVACQParam* par;
371  while ((par = (KVACQParam*)next())) if (par->GetNumber() > maxpar) maxpar = par->GetNumber();
372 
373  TObjArray* parlist = new TObjArray(maxpar, 1);
374  parlist->SetName("ParameterList");
375 
376  next.Reset();
377  while ((par = (KVACQParam*)next())) {
378  parlist->AddAt(new TNamed(par->GetName(), Form("index=%d", par->GetNumber())), par->GetNumber());
379  if (make_arrays) {
380  fUserTree->SetAlias(par->GetName(), Form("Sum$((ParNum==%d)*ParVal)", par->GetNumber()));
381  fUserTree->SetAlias(Form("%s_M", par->GetName()), Form("Sum$(ParNum==%d)", par->GetNumber()));
382  }
383  }
384  fUserTree->GetUserInfo()->Add(parlist);
385 }

410 void KVGANILDataReader::OpenFile(const Char_t* file)
411 {
412  //Open and initialise a GANIL data file for reading.
413  //By default, scaler buffers are ignored.
414  //This can be changed by changing the value of
415  //
416  // KVGANILDataReader.ScalerBuffersManagement: kSkipScaler
417  //
418  //If file cannot be opened, this object will be made a zombie. Do not use.
419  //To test if file is open, use IsOpen().
420  //The basename of the file (excluding any path) can be obtained using GetName()
421  //The full pathname of the file can be obtained using GetTitle()
422  //
423  //If the dataset corresponding to the data to be read is known i.e. if gDataSet has been defined and points
424  //to the correct dataset, the associated multidetector object with the calibration parameters etc.
425  //for the run will be built (any previously existing object pointed to by gMultiDetArray will
426  //be deleted beforehand)
427  //
428  //If not (i.e. if no information is available on detectors, calibrations, geometry, etc.),
429  //then a list of KVACQParam objects will be generated and connected ready for reading the data.
430  //This list can be obtained with method GetRawDataParameters().
431 
432  if (fGanilData) {
433  delete fGanilData;
434  fGanilData = 0;
435  }
436 
437  fGanilData = NewGanTapeInterface();
438 
439  fGanilData->SetFileName(file);
440  SetName(gSystem->BaseName(file));
441  SetTitle(file);
442 
443  // handling scaler buffers
444  TString what_scale = gEnv->GetValue("KVGANILDataReader.ScalerBuffersManagement", "kSkipScaler");
445  what_scale.Prepend("GTGanilData::");
446  Long_t ws = gInterpreter->ProcessLine(what_scale);
447  fGanilData->SetScalerBuffersManagement((GTGanilData::ScalerWhat_t)ws);
448 
449  fGanilData->Open();
450 
451  //test whether file has been opened
452  if (!fGanilData->IsOpen()) {
453  //if initial attempt fails, we try to open the file as a 'raw' TFile
454  //This may work when we are attempting to open a remote file i.e.
455  //via rfio, and a subsequent attempt to open the file using the GanTape
456  //routines may then succeed.
457  TUrl rawtfile(file, kTRUE);
458  rawtfile.SetOptions("filetype=raw");
459  TFile* rawfile = TFile::Open(rawtfile.GetUrl());
460  if (!rawfile) {
461  Error("OpenFile", "File cannot be opened: %s",
462  file);
463  MakeZombie();
464  return;
465  }
466  //TFile::Open managed to open file! Try again...
467  delete rawfile;
468  fGanilData->Open();
469  if (!fGanilData->IsOpen()) {
470  //failed again ??!
471  Error("OpenFile", "File cannot be opened: %s",
472  file);
473  MakeZombie();
474  return;
475  }
476  }
477  // if this data belongs to a known (and currently active dataset), we
478  // build the corresponding multidetector array corresponding to the run
479  if (gDataSet)
480  KVMultiDetArray::MakeMultiDetector(gDataSet->GetName(), fGanilData->GetRunNumber());
481 
482  ConnectRawDataParameters();
483 }

500 void KVGANILDataReader::ConnectRawDataParameters()
501 {
502  //Private utility method called by KVGANILDataReader ctor.
503  //fParameters is filled with a KVACQParam for every acquisition parameter in the file.
504  //If there exists a gMultiDetArray corresponding to this data, we use the KVACQParams
505  //already defined for the detectors of the array whenever possible.
506  //For any parameters for which no KVACQParam already exists (a fortiori if no
507  //gMultiDetArray exists) we create new KVACQParam objects which will be deleted
508  //with this KVGANILDataReader (these objects can be accessed from the list
509  //returned by GetUnknownParameters()).
510  //To access the full list of data parameters in the file after this method has been
511  //called (i.e. after the file is opened), use GetRawDataParameters().
512 
513  TIter next(fGanilData->GetListOfDataParameters());
514  KVACQParam* par;
515  GTDataPar* daq_par;
516  while ((daq_par = (GTDataPar*) next())) {//loop over all parameters
517  par = CheckACQParam(daq_par->GetName());
518  fGanilData->Connect(par->GetName(), par->ConnectData());
519  par->SetNumber(daq_par->Index());
520  par->SetNbBits(daq_par->Bits());
521  fParameters->Add(par);
522  }
523 }

566 Bool_t KVGANILDataReader::GetNextEvent()
567 {
568  // Read next event in raw data file.
569  // Returns false if no event found (end of file).
570  // The list of all fired acquisition parameters is filled, and can be retrieved with
571  // GetFiredDataParameters().
572  // If SetUserTree(TTree*) has been called, the TTree is filled with the values of all
573  // parameters in this event.
574 
575  Bool_t ok = fGanilData->Next();
576  FillFiredParameterList();
577  if (fUserTree) {
578  if (make_arrays) {
579  NbParFired = fFired->GetEntries();
580  TIter next(fFired);
581  KVACQParam* par;
582  int i = 0;
583  while ((par = (KVACQParam*)next())) {
584  ParVal[i] = par->GetCoderData();
585  ParNum[i] = par->GetNumber();
586  i++;
587  }
588  }
589  fUserTree->Fill();
590  }
591  return ok;
592 }
