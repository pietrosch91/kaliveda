18 //KVINDRACodeMask
19 //
20 //Handles bit-masks used to describe identification & calibration codes for particles,
21 //and also used to define which particles are to be considered "ok" or "good" for analysis
22 //(i.e. which particles have IsOK()=kTRUE).
23 //
24 //For compatibility with old VEDA6 codes, a set of bit-mask constants are defined to
25 //describe ID and energy codes:
26 // ID codes: kIDCode0, kIDCode1, ..., kIDCode15
27 // E codes: kECode0, kECode1, ..., kECode15
28 //
29 //_____________________________________________________________________________________
30 //Let us recall the meaning of the VEDA_6.INCL codes :
31 //
32 // code = 0 : gamma
33 // code = 1 : neutron (seulement couronnes 2 a 9)
34 // code = 2 : part. ident. dans CsI ou Phoswich (OK + 8Be)
35 // code = 3 : fragment identifie dans Si-CsI
36 // ou fragment ident. dans Si75-SiLi ou SiLi-CsI
37 // code = 4 : fragment identifie dans ChIo-Si
38 // ou fragment ou part. identifie dans ChIo-Si75
39 // code = 5 : fragment non-ident. (montee de Bragg)(Z min.)
40 // ou stoppe dans Chio (Z min)
41 // code = 6 : frag. cree par la coherence CsI (stoppe ds Si.)
42 // code = 7 : frag. cree par la coherence ChIo (stoppe ds ChIo)
43 // code = 8 : multiple comptage dans ChIo avec arret
44 // - dans les siliciums (cour. 2-9)
45 // - dans les CsI (cour. 10-17)
46 // code = 9 : ident. incomplete dans CsI ou Phoswich (Z.min)
47 // code =10 : ident. "entre les lignes" dans CsI
48 // code =11 : ident. Z=2 due a l'inversion CHIO (Argon)
49 //
50 // code =14 : Particule restee non identifiee (a rejeter)
51 // code =15 : frag. ou part. rejete
52 //
53 // Nota: Pour le code 8 la contribution ChIo est partagee egalement entre
54 // toutes les particules qui s'arretent dans les modules suivants.
55 // Le Z individuel est donc surement faux mais cela permet
56 // d'avoir une assez bonne estimation du Z.total.
57 //
58 // Les codes 9 et 10 sont des codes qui ont ete ajoutes pour donner
59 // au code 2 la notion d'identification sans ambiguites.
60 //____________________________________________________________________________________
61 //
62 //NEW ID CODE MASKS
63 //-----------------
64 //For greater ease of use, we also define new bit-mask constants which are equivalent
65 //to the previous ones, but have names which make their meaning more clear (and easier to
66 //remember) :
67 //
68 // kIDCode_Gamma = kIDCode0,
69 // kIDCode_Neutron = kIDCode1,
70 // kIDCode_CsI = kIDCode2,
71 // kIDCode_Phoswich = kIDCode2,
72 // kIDCode_SiCsI = kIDCode3,
73 // kIDCode_SiLiCsI = kIDCode3,
74 // kIDCode_Si75SiLi = kIDCode3,
75 // kIDCode_ChIoSi = kIDCode4,
76 // kIDCode_ChIoCsI = kIDCode4,
77 // kIDCode_ChIoSi75 = kIDCode4,
78 // kIDCode_Bragg = kIDCode5,
79 // kIDCode_Zmin = kIDCode5,
80 // kIDCode_ZminCsI = kIDCode9,
81 // kIDCode_ArretChIo = kIDCode5,
82 // kIDCode_ArretSi = kIDCode5,
83 // kIDCode_MultihitChIo = kIDCode8
84 //
85 //
86 //DEFINING CODE MASKS
87 //-------------------
88 //Code masks defining acceptable particles for events are defined using the predefined
89 //bit-masks and the bit-wise "OR" as follows:
90 //
91 // SetIDMask( kIDCode2 | kIDCode3 | kIDCode 4 | kIDCode6 ); // ID codes 2-4 & 6 are accepted, all others rejected
92 // SetEMask ( kECode1 | kECode2 ); // accept calibration codes 1 & 2
93 //
94 //
95 //TESTING CODE MASKS
96 //------------------
97 //You can test the identification code of a particle using the TestIDCode method (see
98 //KVINDRAReconNuc on how to obtain the particle's code mask):
99 //
100 // Bool_t idchiosi = TestIDCode( kIDCode_ChIoSi );
101 //
102 //In this example, idchiosi=kTRUE if the particle's ID code indicates detection in a
103 //ChIo-Si telescope.
104 //
105 //This method also works to test if a given code is accepted by the mask set for an
106 //event (see KVINDRAReconEvent).
107 //

208 Bool_t KVINDRACodeMask::TestIDCode(UShort_t code)
209 {
210  //check identification code against code mask
211  //always kTRUE if no mask set (fIDMask==0)
212  if (!fIDMask)
213  return kTRUE;
214  return (Bool_t)((fIDMask & code) != 0);
215 }
