8 // Description
9 //
10 // Wrapper class for the silicon energy minimisation routines. This class allows
11 // the underlying implementation to be changed easily and without affecting any
12 // existing code.
13 //
14 // To change the implementation, for example to use the multi-threaded minimiser
15 // instead of the single threaded one, simply ensure that the appropriate
16 // implementation class header is included and then change any occurrences of
17 // MonoMinimiserImpl in the code to use your chosen implementation (e.g.
18 // ThreadedMinimiserImpl). Then when you recompile KaliVeda the new
19 // implementation will be used instead.
20 //
