16 <h2>KVVarGlob</h2>
17 <h4>Base class for global variables</h4>
18 
19 This class is a base class for the management of global variables.
20 A global variable is an analysis tool for condensing the information in a multibody event into one or a few
21 characteristic values. A simple example is the event multiplicity (the number of particles in each event),
22 which can be used to characterize heavy-ion collision events in terms of violence or centrality.
23 
24 In KaliVeda, the base class for a multibody event is KVEvent, which is basically a collection of
25 nuclei (base class KVNucleus). Therefore the global variable classes below can be used with any
26 event described by a class derived from KVEvent, containing particles described by a class which
27 inherits from KVNucleus.
28 
29 Global variable objects are used in the following schematic way:
30 
31 #### Creation & initialisation:
32 
33 ~~~~~~~~~~~~
34  KVVarGlob* VG = new KV...;
35  VG->Init(); // perform any necessary initialisations
36 ~~~~~~~~~~~~
37 
38 #### Treatment of 1 event:
39 
40 ~~~~~~~~~~~~
41  VG->Reset(); // reinitialise prior to analysis
42  while( [loop over particles in event] ){
43 
44  VG->Fill( [particle] ); // calculate contribution of particle to variable
45  }
46  Double_t valueOfVG = VG->GetValue(); // retrieve value of global variable for event
47 ~~~~~~~~~~~~
48 
49 Global variables can be of different types:
50 
51  - One-body global variable (type = `KVVarGlob::kOneBody`)
52  - the variable is computed by performing a loop over all particles in an event
53  and calling the overridden Fill(KVNucleus*) method for each particle in turn.
54  - Two-body global variable (type = `KVVarGlob::kTwoBody`)
55  - the variable is computed by performing a loop over all pairs of particles in an event
56  and calling the overridden Fill2(KVNucleus*,KVNucleus*) method for each pair in turn.
57  - N-body global variable (type = `KVVarGlob::kNBody`)
58  - the variable is computed from the full list of particles of the event, by defining
59  the overridden FillN(KVEvent*) method.
60 
61 Derived global variable classes of 2-body or N-body type must set the `fType` member variable
62 to the appropriate type (`kTwoBody` or `kNBody`) and define the Fill2(KVNucleus*,KVNucleus*)
63 method (for 2-body variables) or the FillN(KVEvent*) method (for N-body variables).
64 
65 This is handled semi-automatically when using method
66 
67 ~~~~~~~~~~~~
68  MakeClass(const Char_t * classname, const Char_t * classdesc, int type)
69 ~~~~~~~~~~~~
70 
71 to generate a skeleton '.h' and '.cpp' file for the implementation of a new global variable class.
72 
73 By default, global variables are 1-body and must define the Fill(KVNucleus*) method.
74 
75 ## Global variable lists
76 The KVGVList class handles a list of global variables. A list can be used in the following
77 schematic way to calculate several global variables at once:
78 
79 #### Creation & initialisation
80 
81 ~~~~~~~~~~~~
82  KVVGList* VGlist = new KVGVList;
83  VGlist->Add( new KV...("var1") ); // add variable
84  VGlist->Add( new KV...("var2") ); // add variable
85  VGlist->Add( new KV...("var3") ); // add variable
86  ...
87  VGlist->Init(); // initialise all variables
88 ~~~~~~~~~~~~
89 
90 #### Treatment of 1 event
91 
92 ~~~~~~~~~~~~
93  VGlist->CalculateGlobalVariables( [event] ); // calculate contribution of each particle to each variable
94  Double_t valueOfvar1 = VGlist->GetGV("var1")->GetValue(); // retrieve value of "var1" for event
95 ~~~~~~~~~~~~
96 
97 ## Options, parameters, reference frames, particle selection, etc.
98 ### Particle selection
99 The selection of particles which are taken into account can be handled by the variable
100 itself. Define a selection using class KVParticleCondition and then set it by calling
101 method SetSelection(KVParticleCondition&). Then, in your treatment loop for each event,
102 instead of calling the Fill(KVNucleus*) method, call FillWithCondition(KVNucleus*).
103 
104 If you have a list of global variable objects (KVGVList), the KVGVList::Fill method
105 automatically calls FillWithCondition for each KVVarGlob in the list (variables for which
106 no particle selection is defined include all particles).
107 
108 ### Options and parameters
109 In order to give greater flexibility to global variable classes without the need to add
110 member variables and the associated Get/Set methods, we provide methods to handle
111 generic 'options' and 'parameters' for all variables.
112 
113 An 'option' is a name-value pair, the value is a character string. Methods to use are:
114 
115 ~~~~~~~~~~~
116  void SetOption(const Char_t* option, const Char_t* value)
117  Bool_t IsOptionGiven(const Char_t* option)
118  KVString& GetOptionString(const Char_t* option) const
119  void UnsetOption(const Char_t* opt)
120 ~~~~~~~~~~~
121 
122 A 'parameter' is a name-value pair, the value is a double-precision float value. Methods to use are:
123 
124 ~~~~~~~~~~~
125  void SetParameter(const Char_t* par, Double_t value)
126  Bool_t IsParameterSet(const Char_t* par)
127  Double_t GetParameter(const Char_t* par)
128  void UnsetParameter(const Char_t* par)
129 ~~~~~~~~~~~
130 
131 ### Kinematical reference frames
132 For global variables which use kinematical properties of particles, it can be useful to be
133 able to calculate the same variable in different Lorentz reference frames (see KVParticle::SetFrame
134 and KVEvent::SetFrame for how to define and access different frames). Therefore we
135 provide the methods
136 
137 ~~~~~~~~~~~
138  void SetFrame(const Char_t*)
139  const Char_t* GetFrame()
140 ~~~~~~~~~~~
141 
142 which allow to change the reference frame used for the calculation of the variable
143 (depending on the implementation of the specific class).
144 */

469 void KVVarGlob::MakeClass(const Char_t* classname, const Char_t* classdesc, int type)
470 {
471  // Creates skeleton '.h' and '.cpp' files for a new global variable class which
472  // inherits from this class. Give a name for the new class and a short description
473  // which will be used to document the class.
474  //
475  // By default the new class will be of type 1-body. A Fill(KVNucleus*) method
476  // will be generated which the user should complete.
477  // For a 2-body variable, call MakeClass with type = KVVarGlob::kTwoBody.
478  // A skeleton Fill2(KVNucleus*,KVNucleus*) method will be generated.
479  // For a N-body variable, call MakeClass with type = KVVarGlob::kNBody.
480  // A skeleton FillN(KVEvent*) method will be generated.
481 
482  // basic class template
483  KVClassFactory cf(classname, classdesc, "KVVarGlob", kTRUE);
484 
485  KVString body;
486 
487  // add 'init' method
488  KVVarGlob::AddInitMethod(classname, cf, body, type);
489 
490  // add 'Fill', 'Fill2', or 'FillN' method
491  KVVarGlob::AddFillMethod(cf, type);
492 
493  // body of 'Fill', 'Fill2', or 'FillN' method
494  KVVarGlob::FillMethodBody(body, type);
495 
496  // add body of method
497  KVVarGlob::AddFillMethodBody(cf, body, type);
498 
499  cf.GenerateCode();
500 }

508 void KVVarGlob::FillMethodBody(KVString& body, int type)
509 {
510  // PRIVATE method used by MakeClass.
511  // body of 'Fill', 'Fill2', or 'FillN' method
512  switch (type) {
513  case kTwoBody:
514  body = " // Calculation of contribution to 2-body global variable of pair (n1,n2) of nuclei.\n";
515  body += " // NOTE: this method will be called for EVERY pair of nuclei in the event\n";
516  body += " // (i.e. n1-n2 and n2-n1), including pairs of identical nuclei (n1 = n2).\n";
517  body += " // If you want to calculate a global variable using only each non-identical pair once,\n";
518  body += " // then make sure in your implementation that you check n1!=n2 and divide the\n";
519  body += " // contribution to any sum by 2 to avoid double-counting.\n";
520  break;
521  case kNBody:
522  body = " // Calculation of contribution to N-body global variable of particles in event e.\n";
523  break;
524  default:
525  body = " // Calculation of contribution to 1-body global variable of nucleus n\n";
526  }
527 }

535 void KVVarGlob::AddInitMethod(const Char_t* classname, KVClassFactory& cf, KVString& body, int type)
536 {

568 void KVVarGlob::AddFillMethod(KVClassFactory& cf, int type)
569 {

571  // add 'Fill', 'Fill2', or 'FillN' method
572  switch (type) {

576  cf.AddMethodArgument("Fill2", "KVNucleus*", "n2");
577  break;
578  case kNBody:
579  cf.AddMethod("FillN", "void");
580  cf.AddMethodArgument("FillN", "KVEvent*", "e");
581  cf.AddHeaderIncludeFile("KVEvent.h");
582  break;
583  default:
584  cf.AddMethod("Fill", "void");
585  cf.AddMethodArgument("Fill", "KVNucleus*", "n");
