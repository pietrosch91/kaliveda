21 <h2>KVIntegerList</h2>
22 <h4>Permet de gerer une liste de nombres entiers positifs</h4>
23 La classe dérive de TNamed, le champ TNamed::fName contient le nom formaté de la partition et TNamed::fTitle <br>
24 permet de stocker la population <br>
25 L'écriture/lecture dans un fichier .root se fait en utilisant le streamer de TNamed <br>
26 Les valeurs sont triées par ordre décroissant avec gestion de l'occurence <br>
27 Les deux exemples suivants montrent les deux facons de remplir une liste: <br>
28 <h5>Exemple 1:</h5>
29 Les methodes de type Fill(), remplissent en une fois la liste et mets à jour les grandeurs reliées<br>
30 <br>
31 <code>
32 KVIntegerList* ilist = new KVIntegerList();<br>
33 Int_t tab[10]={1,3,6,4,9,7,3,59,8,160};<br>
34 ilist->Fill(tab,10);<br>
35 ilist->Print();<br>
36 Info in <KVIntegerList::Print>: 160 59 9 8 7 6 4 3(2) 1 : population 1<br>
37 </code>
38 <h5>Exemple 2:</h5>
39 Les methodes de type Add(), remplit successivement la liste, avant d'utiliser les grandeurs reliées<br>
40 l'utilisateur doit appeler la méthode CheckForUpdate(), pour que la mise a jour du nom soit faite<br>
41 par contre le nombre d'elements est mis a jour automatiquement<br>
42 <br>
43 <code>
44 KVIntegerList* ilist = new KVIntegerList();<br>
45 ilist->Add(23,6);<br>
46 ilist->Add(12);<br>
47 Int_t tab[3]={1,3,6};<br>
48 ilist->Add(tab,3);<br>
49 ilist->CheckForUpdate();<br>
50 ilist->Print();<br>
51 Info in <KVIntegerList::Print>: 23(6) 12 6 3 1 : population 1<br>
52 ilist->Remove(23,3);<br>
53 ilist->Print();<br>
54 Info in <KVIntegerList::Print>: 23(6) 12 6 3 1 : population 1 &nbsp &nbsp !!! pour le nom RIEN A CHANGER<br>
55 <code>ilist->GetNbre();<br>
56 (const Int_t)7 &nbsp &nbsp &nbsp &nbsp &nbsp &nbsp &nbsp &nbsp le nombre, lui est mis à jour<br>
57 ilist->CheckForUpdate();<br>
58 ilist->Print();<br>
59 Info in <KVIntegerList::Print>: 23(3) 12 6 3 1 : population 1 &nbsp &nbsp OK : les modifs ont été prises en compte<br>
60 ilist->GetNbre()<br>
61 (const Int_t)7
62 </code>
63 END_HTML
64 
65 Les methodes de type Remove(), retirent des valeurs de la liste, là encore l'utilisateur doit appeler
66 la méthode CheckForUpdate(), pour que la mise a jour soit faite
67 Les grandeurs disponibles sont:
68 - le nombre d'entiers, ie la multiplicité GetNbre()
69 - le nom de la partition definie comme suit : ent1(occ) ent2 ent3(occ3), où entre parenthèses
70 sont notées les occurences aupérieures à 1 d'une valeur
71 - la population GetPopulation(), permet de gérer un ensemble de partitions (KVPartitionManager)

236 void KVIntegerList::SetPartition(const Char_t* par)
237 {
238 //protected method, utilisée par le Streamer qui utilise le champ fName de la classe TNamed
239 //voir également KVIntegerList::DeducePartitionFromTNamed
240  KVString st(par);
241  st.Begin(" ");
242  TObjArray* toks = 0;
243  while (!st.End()) {
244  KVString tamp = st.Next();
245  Int_t val;
246  Int_t freq;
247  if (tamp.Contains("(")) {
248  if (!tamp.Contains(")")) {
249  Warning("SetPartition", "%s ->pb de coherence dans les parentheses", tamp.Data());
250  return;
251  } else {
252  toks = tamp.Tokenize("(");
253  val = ((TObjString*)toks->At(0))->GetString().Atoi();
254  freq = ((TObjString*)toks->At(1))->GetString().Atoi();
255  delete toks;
256  }
257  } else {
258  val = tamp.Atoi();
259  freq = 1;
260  }
261  Add(val, freq);
262  }

517 Bool_t KVIntegerList::remove_values(Int_t val, Int_t freq)
518 {
519 //protected method, On retire "freq" fois la valeur val
520 //la methode retourne kTRUE si cette valeur etait effectivement presente, kFALSE sinon
521 //Si freq > la frequence initiale ( KVIntegerList::GetFrequency(Int_t ) )de la valeur, on la retire complétement
522 //Le bit kHastobeComputed es mis à 1 pour indiquer la nécéssité de mettre a jour la partition
523 //voir KVIntegerList::CheckForUpdate()
524 //Cette methode est appelée par les autres routines de type Remove...(...), c'est celle-ci qui doit etre derivée

529 
530  if (val > fLimiteRegle) return kFALSE;

535  fRegle->AddAt(TMath::Max(fRegle->At(val) - freq, 0), val);
