10 //KVINDRACodes
11 //
12 //Describes the identification and calibration status of reconstructed particles.
13 //Based on the codes figuring in veda_6.incl of the old VEDA programme.
14 //
15 //GetVedaIDCode() - returns ID code in VEDA format (0 = gamma, 2 = id CsI/Phoswich, etc.)
16 //GetVedaECode() - returns energy calibration code in VEDA format (1 = calibration sans probleme, etc.)
17 //GetIsotopeResolve() returns kTRUE if the particle's mass was identified, kFALSE if the mass is calculated from Z.
18 // IN BOTH CASES KVINDRAReconNuc::GetA() RETURNS A>0
19 //
20 //GetIDStatus() - returns a string describing the identification status
21 // e.g. if GetVedaIDCode()=0 then GetIDStatus()="gamma"
22 //GetEStatus() - returns a string describing the calibration status
23 // e.g. if GetVedaECode()=0 then GetEStatus()="Aucun appel a une routine de calibration effectuee"
24 //
25 //WARNING: the methods GetIDCode() and GetECode() return the new bit-mask status codes defined in
26 // class KVINDRACodeMask.
27 //If you prefer to use these masks, the best way to test the status of the particles is by
28 //using KVINDRACodeMask::TestIDCode and KVINDRACodeMask::TestECode
29 //
30 //e.g. to test if a particle was identified using CsI R-L you have three choices
31 //(supposing the particle is referenced by KVINDRAReconNuc* part):
32 //
33 // EITHER if( part->GetCodes().GetVedaIDCode() == 2 ) { ... }
34 // OR if( part->GetCodes().TestIDCode( kIDCode2 ) ) { ... }
35 // OR if( part->GetCodes().TestIDCode( kIDCode_CsI ) ) { ... ]
36 //
37 //Obviously, the third method is to be preferred (it is immediately clear what the instruction means)!!
38 //

282 UChar_t KVINDRACodes::GetCodeIndex(UShort_t mask)
283 {
284  //Argument is a bit-mask representing particle ID or E code
285  //Returns the number of the bit which is set to 1 (right-most bit is 0).
286  //Result is corresponding index in arrays fCodeGenIdent, fIDCodes, etc.
287  int i = 0;
288  if (!mask)
289  return 0;
290  do {
291  mask = (mask >> 1);
292  i++;
293  } while (mask);
294  return i - 1;
295 }
