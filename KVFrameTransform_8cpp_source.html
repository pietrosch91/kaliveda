11 <h2>KVFrameTransform</h2>
12 <h4>Utility class for kinematical transformations of KVParticle class</h4>
13 
14 ~~~~~~~~~~
15  N.B.: for boosts, default units are cm/ns
16  for rotations (TRotation), default units are radians
17 ~~~~~~~~~~
18 
19  This class is used as a generic argument for the KVEvent::SetFrame and
20  KVParticle::SetFrame methods (amongst others). It is not always necessary to
21  explicitly use this class when calling these methods. Any class for which
22  a single-argument KVFrameTransform constructor is defined can be used, even
23  a temporary (unnamed) object:
24 
25  e.g. for KVParticle::SetFrame(const Char_t* frame, const KVFrameTransform& ft)
26  the following calls are valid:
27 
28 ~~~~~~~~~~
29  KVParticle p;
30  TVector3 boost(0,0,3.0);// in cm/ns
31  p.SetFrame("toto", boost);
32  p.SetFrame("toto", TVector3(0,0,3.0)); // same effect as previous
33  TRotation rot;
34  rot.RotateZ(TMath::PiOver2()); // 90 deg. rotation around beam
35  p.SetFrame("titi", rot);
36 ~~~~~~~~~~
37 
38  On the other hand, when more than one argument to the KVFrameTransform constructor
39  is required, you have to explicitly call the constructor in question, although
40  once again it is sufficient to use a temporary object in the method call:
41 
42 ~~~~~~~~~~
43  p.SetFrame("tata", KVFrameTransform(TVector3(0,0,0.1),kTRUE));
44  --> boost to frame moving at 0.1c in beam direction
45 
46  p.SetFrame("tutu", KVFrameTransform(rot, TVector3(0,0,0.1), kTRUE));
47  --> rotation around beam axis followed by boost 0.1c
48 ~~~~~~~~~~
49 
50 */

104 KVFrameTransform::KVFrameTransform(const TVector3& boost, Bool_t beta)
105  : TLorentzRotation()
106 {
107  // Construct frame transformation using velocity boost vector
108  // if beta=kTRUE, velocity given in light speed units
109  // if beta=kFALSE [default], velocity given in cm/ns units.
110  if (beta) {
111  Boost(boost);
112  } else {
113  Boost(boost.X() / KVParticle::C(), boost.Y() / KVParticle::C(),
114  boost.Z() / KVParticle::C());
115  }
