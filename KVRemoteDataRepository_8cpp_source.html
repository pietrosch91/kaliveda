20 <h2>KVRemoteDataRepository</h2>
21 <h4>Class for accessing a collection of data files on a remote machine</h4>
22 <p>
23 Data repositories are defined in the user's $HOME/.kvrootrc; some typical examples are given
24 in $KVROOT/KVFiles/.kvrootrc:
25 </p>
26 <pre>
27 DataRepository: ccali
28 ccali.DataRepository.Type: remote
29 ccali.DataRepository.RootDir: cchpssindra:/hpss/in2p3.fr/group/indra
30 ccali.DataRepository.ReadProtocol: root
31 ccali.DataRepository.XRDServer: ccxrdsn012:1999
32 ccali.DataRepository.XRDRootDir: /hpss/in2p3.fr/group/indra
33 ccali.DataRepository.XRDTunnel.host: ccali.in2p3.fr
34 ccali.DataRepository.XRDTunnel.port: 10000
35 ccali.DataRepository.XRDTunnel.user:
36 ccali.DataRepository.RemoteAvailableRuns.protocol: curl
37 ccali.DataRepository.RemoteAvailableRuns.url: http://indra.in2p3.fr/KaliVedaDoc
38 ccali.DataRepository.FileTransfer.type: bbftp
39 ccali.DataRepository.FileTransfer.server: ccbbftp.in2p3.fr
40 ccali.DataRepository.FileTransfer.user:
41 </pre>
42 <p>
43 This is the definition of a "remote" data repository.
44 A remote data repository is principally characterised by the fact that access to the data files is
45 via some non-local protocol: in this case it is xrootd ("...ReadProtocol: root"). The host name and port of
46 the xrootd server are given ("...XRDServer: ccxrdsn012:1999"), as well as the root directory
47 to be used ("...XRDRootDir: /hpss/in2p3.fr/group/indra").
48 </p>
49 <p>
50 In order to use a remote repository, some way to access the database of available
51 data files for each run type must be defined. Here it is the curl programme which is used to read
52 them via a website ("...RemoteAvailableRuns.protocol: curl"; "...RemoteAvailableRuns.url: http://indra.in2p3.fr/KaliVedaDoc").
53 </p>
54 <p>
55 Another particularity of remote data repositories is that some additional stuff may be needed
56 in order to be able to access the data.
57 This is handled by <a href="KVRemoteDataRepository.html#KVRemoteDataRepository:IsConnected">KVRemoteDataRepository::IsConnected()</a>.
58 In the present example, an SSH tunnel is set up in order to allow secure connection to the
59 xrootd server ("...XRDTunnel.*:").
60 </p>
61 <p>
62 Finally, transfer of data between data repositories is handled by <a href="KVDataTransfer.html">KVDataTransfer</a> and child classes.
63 In the present example, this is configured to use bbftp ("...FileTransfer.type: bbftp"), for which the required details are given.
64 </p>

344 const Char_t* KVRemoteDataRepository::GetFullPathToTransferFile(const KVDataSet* dataset,
345  const
346  Char_t*
347  datatype,
348  const
349  Char_t*
350  runfile)
351 {
352  //Used by KVDataTransfer.
353  //Returns the full path needed to transfer a runfile belonging to the given dataset
354  //either from or to the repository, using sftp or bbftp etc.
355  //This is a concatenation of the repository root directory with the dataset
356  //subdirectories and filename.
357  //However, for a remote data repository, the type of operating system may be different
358  //to gSystem, i.e. the system on the local machine. In this case, gSystem->ConcatFileName
359  //may not necessarily give the right paths for the remote machine: e.g. local machine
360  //is Windows, remote repository is Linux, paths for Linux machine will have a mix of
361  //"/" and "\" in them.
362  //We try to remedy this by looking at fLocalrootdir: if it contains "/" we take the result
363  //of the concatenation performed by KVDataRepository::GetFullPathToTransferFile
364  //and replace any "\" by "/", and vice versa.
365 
366  static TString path;
367  path =
368  KVDataRepository::GetFullPathToTransferFile(dataset, datatype,
369  runfile);
370  if (fLocalrootdir.Contains("/"))
371 // path.ReplaceAll("\\", "/");
372  path = gSystem->UnixPathName(path.Data());
373  else if (fLocalrootdir.Contains("\\"))
374  path.ReplaceAll("/", "\\");
375  return path.Data();
376 }

389 const Char_t* KVRemoteDataRepository::GetFullPathToOpenFile(const KVDataSet* dataset,
390  const Char_t*
391  datatype,
392  const Char_t*
393  runfile)
394 {
395  //When accessing remote data repositories, the operating system corresponding
396  //to the remote file system may not be the same as the local operating system
397  //(corresponding to gSystem). Therefore use of gSystem->ConcatFileName may lead
398  //to paths which mix '/' and '\'.
399  //We check for this here by converting the result of KVDataRepository::GetFullPath
400  //depending on whether the root directory of this repository contains '/' or '\'
401  static TString path;
402  path =
403  KVDataRepository::GetFullPathToOpenFile(dataset, datatype,
404  runfile);
405  if (fLocalrootdir.Contains("/"))
406  path = gSystem->UnixPathName(path.Data());
407  else if (fLocalrootdir.Contains("\\"))
408  path.ReplaceAll("/", "\\");
409  return path.Data();
410 }

434 Bool_t KVRemoteDataRepository::IsConnected()
435 {
436  //Before reading data in a remote data repository, it may be necessary to 'connect' to the repository
437  //in some way: e.g. we may need to open an SSH tunnel to an xrootd server.
438  //Call this method to make sure any required connection has been made, and if successful
439  //the method returns kTRUE; otherwise return kFALSE.
440  //
441  //XROOTD via SSH tunnel
442  //===================
443  //Configure using the following environment variables:
444  //
445  // ccali.DataRepository.XRDTunnel.host: ccali.in2p3.fr
446  // ccali.DataRepository.XRDTunnel.port: 10000
447  //
448  //The host and port to use are obligatory. Optionally you may also change the following:
449  //
450  // ccali.DataRepository.XRDTunnel.retry: #max number of tries before giving up on connection, default = 30
451  //
452  // ccali.DataRepository.XRDTunnel.user: #username for login to tunnel host. default is local username.
453 
454  if (fXRDtunnel) {
455  //check existence of SSH tunnel to xrootd server
456  TSocket* sock = new TSocket("localhost", fXRDtunPort);
457  Int_t iRetry = 0;
458  if (!sock->IsValid()) {
459  //open tunnel
460  TString command = "xterm -e \"ssh -X -L";
461  command += fXRDtunSpec;
462  command += " ";
463  command += Form("%s@%s\" &", fXRDtunUser.Data(), fXRDtunHost.Data());
464  gSystem->Exec(command.Data());
465  cout << "Waiting for connection to xrootd server via SSH tunnel ";
466  while (!sock->IsValid() && iRetry < fXRDtunRetry) {
467  cout << ".";
468  cout.flush();
469  gSystem->Sleep(2000);
470  sock->Close();
471  delete sock;
472  sock = new TSocket("localhost", fXRDtunPort);
473  iRetry++;
474  }
475  cout << endl;
476  }
477  sock->Close();
478  delete sock;
479  if (iRetry == fXRDtunRetry) {
480  cout << "Connection timeout " << endl;
481  return kFALSE;
482  }
483  }
484  return kTRUE;
485 }
