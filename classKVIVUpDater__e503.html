IMPORTANT!

An exception can be thrown here (intentionally, as one can't pass a return code back up the stack). The reason for this is that the function signature must match that of KVINDRAUpDater::SetParameters (which has a void return type).

The advantage of throwing an exception is that the error can be handled properly further up the stack in KVIVSelector::Notify. KVIVSelector::Notify will automatically end the current run if it catches this exception.

The alternative is the unexpected termination of the parent process, which if you happen to be in an interactive ROOT session is bad news (it will kill that too).
