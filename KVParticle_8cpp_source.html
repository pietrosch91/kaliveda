35 /*
37 <h2>KVParticle</h2>
38 <h4>A general base class for relativistic kinematics of massive particles</h4>
39 
40 Implements all kinematical manipulations necessary for obtaining angles, kinetic energies, transverse energies etc. etc.,
41 defining and changing relativistic reference frames.
42 
43 Unless otherwise stated,
44  - all energies are in MeV
45  - all velocities are in cm/ns
46  - all angles are in degrees (polar angles between 0 and \f$180^o\f$; azimuthal angles between 0 and \f$360^o\f$).
47  - all momenta are in MeV/c
48 
49 This class derives from [TLorentzVector](https://root.cern.ch/doc/master/classTLorentzVector.html) - a particle is basically a Lorentz 4-vector with added attributes - and therefore
50 any methods which are not documented here will be found in that class
51  - [**N.B.** most 'Set'ter methods of TLorentzVector should not be used directly and have been made 'private' in KVParticle in order to prevent their use].
52 
53 This also means that all particles and kinematics are relativistic in KaliVeda.
54 
55 Methods defined/redefined in this class:
56 
57 ~~~~~~~~~~~~~~~~~~
58  GetMass() return mass of particle in MeV/c**2 (same as TLorentzVector::M())
59  GetMomentum() returns momentum 3-vector (TVector3) of particle (units MeV/c) (same as TLorentzVector::Vect())
60  GetKE()/GetEnergy() returns kinetic energy in MeV (same as TLorentzVector::E()-TLorentzVector::M())
61  GetTransverseEnergy()/GetEtran() see below
62  GetRTransverseEnergy()/GetREtran() see below
63  GetVelocity()/GetV() returns velocity 3-vector (TVector3) of particle (units cm/ns)
64  GetVpar() returns velocity component in beam (z) direction (cm/ns) (same as GetV().Z())
65  GetVperp() same as GetV().Perp(), but sign is same as y-component of velocity, GetV().Y()
66  GetTheta() same as TLorentzVector::Theta() but in degrees, not radians
67  GetPhi() same as TLorentzVector::Phi() but in degrees, not radians, and always positive, between 0 and TMath::TwoPi()
68 ~~~~~~~~~~~~~~~~~~
69 
70 Particle properties can be defined either using one of the constructors :
71 
72 ~~~~~~~~~~~~~~~~~~
73  KVParticle(Double_t m, TVector3 & p); // rest mass and momentum 3-vector
74  KVParticle(Double_t m, Double_t px, Double_t py, Double_t pz); // rest mass and Cartesian components of momentum 3-vector
75 ~~~~~~~~~~~~~~~~~~
76 
77 or with the usual 'Set' methods:
78 
79 ~~~~~~~~~~~~~~~~~~
80  SetMass(Double_t m); // changes rest mass, leaves momentum unchanged
81  SetEnergy(Double_t e); // set kinetic energy (in MeV)
82  SetMomentum(const TVector3 & v); // changes momentum, leaves rest mass unchanged
83  SetMomentum(const TVector3 * v);
84 ~~~~~~~~~~~~~~~~~~
85 
86 or, for the following method:
87 
88 ~~~~~~~~~~~~~~~~~~
89  SetMomentum(Double_t px, Double_t py, Double_t pz, Option_t * opt ="cart");
90 ~~~~~~~~~~~~~~~~~~
91 
92 Set momentum 3-vector, leaving rest mass unchanged, with :
93  - if `opt = "cart"` or `"cartesian"` : using cartesian components `(px,py,pz)`
94  - if `opt = "spher"` or `"spherical"` : set momentum 3-vector using spherical coordinates
95  - in this case `px` = magnitude, `py` = theta [degrees], `pz` = phi [degrees].
96 
97  Other methods:
98 
99 ~~~~~~~~~~~~~~~~~~
100  SetMomentum(Double_t T, TVector3 dir); // set kinetic energy to T [MeV] and direction given by 'dir' unit vector
101 
102  SetRandomMomentum(Double_t T, Double_t thmin, Double_t thmax, Double_t phmin, Double_t phmax, Option_t * opt = "isotropic");
103  // a handy tool for giving random momenta to particles.
104 ~~~~~~~~~~~~~~~~~~
105 
106 GetEtran() and GetTransverseEnergy() return the non-relativistic transverse energy defined as the kinetic energy multiplied by the squared sinus of the polar angle:
107 i.e. \f$E_{tran} = E\sin^2(\theta)\f$ where \f$\theta\f$ is the polar angle of the particle measured with respect to the beam (z-) axis.
108 
109 GetREtran() and GetRTransverseEnergy() return the relativistic transverse energy which is frame invariant.
110 
111 ## Kinematical reference frames
112 ### 1. Accessing particle kinematics in different frames
113 Particle kinematics in different frames can be accessed with InFrame(const KVFrameTransform&).
114 This method does not modify the particle's kinematics or default reference frame (see 2 below).
115 If you want to access lots of information from this frame, it is probably more efficient to
116 define and store it in the particle's list of reference frames (see 3 below).
117 
118 __Example:__ inspect kinematics of particle (accessed through pointer `KVParticle* p`)
119  - in a frame moving at 5 cm/ns in the beam direction (velocities/boosts are described by [TVector3](https://root.cern.ch/doc/master/classTVector3.html) objects):
120 
121 ~~~~~~~~~~~~~~~~~~
122  p->InFrame(TVector3(0,0,5)).GetVpar();
123 ~~~~~~~~~~~~~~~~~~
124 
125  - in a frame rotated by \f$90^o\f$ clockwise around the +ve beam direction (rotations are described by [TRotation](https://root.cern.ch/doc/master/classTRotation.html) objects):
126 
127 ~~~~~~~~~~~~~~~~~~
128  TRotation rot;
129  rot.RotateZ(TMath::PiOver2());
130  p->InFrame(rot).GetPhi();
131 ~~~~~~~~~~~~~~~~~~
132 
133  - in a frame moving at \f$0.1c\f$ in the beam direction:
134 
135 ~~~~~~~~~~~~~~~~~~
136  p->InFrame(KVFrameTransform(TVector3(0,0,0.1),kTRUE)).GetKE();
137 
138  // the following only works with C++11 and later
139  p->InFrame({{0,0,0.1},kTRUE}).GetKE();
140 ~~~~~~~~~~~~~~~~~~
141 
142 In this case we need to pass two arguments to the KVFrameTransform(const TVector3&, Bool_t = kFALSE) constructor;
143 before C++11, this required an explicit call to the constructor.
144 
145 ### 2. Modifying particle kinematics
146 Particle kinematics can be modified using method ChangeFrame(const KVFrameTransform&):
147 
148 __Example:__ change kinematics of particle (accessed through pointer `KVParticle* p`)
149  - to a frame moving at 5 cm/ns in the beam direction:
150 
151 ~~~~~~~~~~~~~~~~~~
152  p->ChangeFrame(TVector3(0,0,5));
153 ~~~~~~~~~~~~~~~~~~
154 
155  - to a frame rotated by \f$90^o\f$ clockwise around the +ve beam direction:
156 
157 ~~~~~~~~~~~~~~~~~~
158  TRotation rot;
159  rot.RotateZ(TMath::PiOver2());
160  p->ChangeFrame(rot);
161 ~~~~~~~~~~~~~~~~~~
162 
163  - to a frame moving at \f$0.1c\f$ in the beam direction:
164 
165 ~~~~~~~~~~~~~~~~~~
166  p->ChangeFrame(KVFrameTransform(TVector3(0,0,0.1),kTRUE));
167 
168  // the following only works with C++11 and later
169  p->ChangeFrame({{0,0,0.1},kTRUE});
170 ~~~~~~~~~~~~~~~~~~
171 
172 ### 3. Using several reference frames
173 Rather than changing the reference frame of the particle, you can define and use several
174 different reference frames while keeping the original kinematics as the default. Each
175 frame can be used independently, and new frames can be defined based on any of the
176 existing frames:
177 
178 __Example:__ (for a particle accessed through pointer `KVParticle* p`):
179  - define a new frame moving at 5 cm/ns in the beam direction:
180 
181 ~~~~~~~~~~~~~~~~~~
182  p->SetFrame("moving_frame", TVector3(0,0,5));
183 ~~~~~~~~~~~~~~~~~~
184 
185  - define a rotated coordinate frame in the "moving_frame", rotated by \f$90^o\f$ clockwise around the +ve beam direction:
186 
187 ~~~~~~~~~~~~~~~~~~
188  TRotation rot;
189  rot.RotateZ(TMath::PiOver2());
190  p->SetFrame("rotated_moving_frame", "moving_frame", rot);
191 ~~~~~~~~~~~~~~~~~~
192 
193  Note that the same frame can be defined directly from the original particle by using a combined boost-then-rotation transform:
194 
195 ~~~~~~~~~~~~~~~~~~
196  p->SetFrame("rotated_moving_frame", KVFrameTransform(TVector3(0,0,5),rot));
197 
198  // the following only works with C++11 and later
199  p->SetFrame("rotated_moving_frame", {{0,0,5},rot});
200 ~~~~~~~~~~~~~~~~~~
201 
202  - define a similarly rotated coordinate frame in the original (default) reference frame:
203 
204 ~~~~~~~~~~~~~~~~~~
205  p->SetFrame("rotated_frame", rot);
206 ~~~~~~~~~~~~~~~~~~
207 
208  - access kinematical information in any of these frames:
209 
210 ~~~~~~~~~~~~~~~~~~
211  p->GetFrame("moving_frame")->GetVpar();
212  p->GetFrame("rotated_frame")->GetPhi();
213  p->GetFrame("rotated_moving_frame")->GetTransverseEnergy();
214 ~~~~~~~~~~~~~~~~~~
215 
216 Note that the frame `"rotated_moving_frame"` is directly accessible even if it is defined in two
217 steps as a rotation of the `"moving_frame"`.
218 
219 ### 4. Changing the default kinematics
220 Let us consider a particle for which the different reference frames in the previous paragraph have been defined.
221 For an example, imagine that the default kinematics are that of particle with rest mass 939 MeV moving with
222 a momentum of 250 MeV/c at an angle of \f$45^o\f$ to the +ve z-direction:
223 
224 ~~~~~~~~~~~~~~~~~~
225 KVParticle p(939,0,0,250);
226 p.SetTheta(45);
227 
228 p.SetFrame("moving_frame", TVector3(0,0,5));
229 TRotation rot;
230 rot.RotateZ(TMath::PiOver2());
231 p.SetFrame("rotated_moving_frame", "moving_frame", rot);
232 p.SetFrame("rotated_frame", rot);
233 ~~~~~~~~~~~~~~~~~~
234 
235 Calling Print() will show all reference frames defined for the particle:
236 
237 ~~~~~~~~~~~~~~~~~~
238 p.Print()

244 ~~~~~~~~~~~~~~~~~~
245

247 The first line is the default kinematics. As yet it has no name:

294 ### 5. Updating stored kinematical frames
295 If you call SetFrame() several times with the same frame name
296 [note that frame names are **case insensitive**], the existing reference frame will
297 be updated to use the new transformation, which will be applied to the kinematics
298 of the particle in the 'parent' frame used to define the frame. Any frames which
299 were defined based on the frame will be updated too.
300 
301 __Example:__ for the previous particle & frame definitions, after resetting the default kinematics:
302 
303 ~~~~~~~~~~~~~~~~~~
304  p.ChangeDefaultFrame("lab");
305 ~~~~~~~~~~~~~~~~~~
306 
307  - change the angle of rotation in the moving rotated frame:
308 
309 ~~~~~~~~~~~~~~~~~~
310  rot.RotateZ(-TMath::PiOver4());
311  p.SetFrame("rotated_moving_frame", rot);
312  p.Print();
313 
314 KVParticle mass=939 Theta=45 Phi=0 KE=32.7103 Vpar=5.45392
315  rotated_frame: Theta=45 Phi=270 KE=32.7103 Vpar=5.45392
316  moving_frame: Theta=85.1751 Phi=0 KE=16.6117 Vpar=0.468125
317  rotated_moving_frame: Theta=45 Phi=315 KE=32.7103 Vpar=5.45392
318 KVNameValueList::ParticleParameters : Parameters associated with a particle in an event (0x7faae41241b8)
319  <frameName=lab>
320 ~~~~~~~~~~~~~~~~~~
321 
322  - change the velocity of the moving frame to \f$0.1c\f$:
323 
324 ~~~~~~~~~~~~~~~~~~
325  p.SetFrame("moving_frame", KVFrameTransform(TVector3(0,0,0.1),kTRUE));

327  // the following only works with C++11 and later

366 ## Definition of groups
367 AddGroup() and BelongsToGroup() methods allow to sort particles in an event into subsets
368 based on various criteria such as "emitted by QP", "backwards emitted", or "include in calorimetry".
369 The group attribution can be done 'by hand' or using a KVParticleCondition object to define a selection.
370 The number of such groups for a given particle is unlimited.
371 
372 Group names are case _insensitive_. Example:
373 
374 ~~~~~~~~~~~~~~
375 KVNucleus nn;
376 nn.AddGroup("forward");
377 nn.BelongsToGroup("ForWaRD");// -> return kTRUE
378 ~~~~~~~~~~~~~~
379 
380  Any groups added to the particle are propagated to all existing kinematical frames. In other words, if a particle has a kinematical frame `"CM"` defined when it is
381  added to a group:
382 
383 ~~~~~~~~~~~~~~~
384 part.SetFrame("CM", [some transformation]);
385 part.AddGroup("toto");
386 part.BelongsToGroup("toto");// this returns kTRUE
387 part.GetFrame("CM")->BelongsToGroup("toto");// this returns kTRUE also
388 ~~~~~~~~~~~~~~~
389 
390 On the contrary if some change is made in the following way :
391 
392 ~~~~~~~~~~~~~~~
393 part.GetFrame("CM")->AddGroup("titi");
394 part.GetFrame("CM")->BelongsToGroup("titi");// this returns kTRUE
395 part.BelongsToGroup("titi");// this returns kFALSE
396 ~~~~~~~~~~~~~~~
397

867 void KVParticle::SetRandomMomentum(Double_t T, Double_t thmin,
868  Double_t thmax, Double_t phmin,
869  Double_t phmax, Option_t* opt)
870 {
871  //Give randomly directed momentum to particle with kinetic energy T
872  //Direction will be between (thmin,thmax) [degrees] limits in polar angle,
873  //and (phmin,phmax) [degrees] limits in azimuthal angle.
874  //
875  //If opt = "" or "isotropic" (default) : direction is isotropically distributed over the solid angle
876  //If opt = "random" : direction is randomly distributed over solid angle
877  //
878  //Based on KVPosition::GetRandomDirection().
879 
880  Double_t p = (T + M()) * (T + M()) - M2();
881  if (p > 0.)
882  p = (TMath::Sqrt(p)); // calculate momentum
883  else
884  p = 0.;
885 
886  TVector3 dir;
887  KVPosition pos(thmin, thmax, phmin, phmax);
888  dir = pos.GetRandomDirection(opt); // get isotropic unit vector dir
889  if (p && dir.Mag())
890  dir.SetMag(p); // set magnitude of vector to momentum required
891  SetMomentum(dir); // set momentum 4-vector
892 }

900 void KVParticle::SetMomentum(Double_t T, TVector3 dir)
901 {
902  //set momentum with kinetic energy t and unit direction vector d
903  //(d is normalised first in case it is not a unit vector)
904 
905  Double_t p = (T + M()) * (T + M()) - M2();
906  TVector3 pdir;
907  TVector3 unit_dir = dir.Unit();
908  if (p > 0.) {
909  p = (TMath::Sqrt(p));
910  pdir = p * unit_dir;
911  }
912  SetMomentum(pdir);
913 };

961 void KVParticle::SetKE(Double_t ecin)
962 {
963  //Change particle KE, keeping momentum direction constant
964  //If momentum is zero (i.e. no direction defined) the particle will be given
965  //a velocity in the positive z-direction
966 
967  if (ecin > 0.) {
968  Double_t et = M() + ecin; // new total energy = KE + m
969  Double_t pmod = 0;
970  if (et * et > M2()) {
971  pmod = TMath::Sqrt(et * et - M2()); // p**2 = E**2 - m**2
972  TVector3 newp(Px(), Py(), Pz());
973  if (pmod && newp.Mag()) {
974  newp.SetMag(pmod);
975  } else {
976  newp.SetXYZ(0, 0, 1);
977  newp.SetMag(pmod);
978  }
979  SetMomentum(newp);
980  } else {
981  SetMomentum(0., 0., 0.);
982  }
983  } else
984  SetMomentum(0., 0., 0.);
985 }

1457 void KVParticle::ChangeDefaultFrame(const Char_t* newdef, const Char_t* defname)
1458 {
1459  // Make existing reference frame 'newdef' the new default frame for particle kinematics.
1460  // The current default frame will then be accessible from the list of frames
1461  // using its name (if set with SetFrameName). You can change/set the name of the previous
1462  // default frame with 'defname'
1463 
1464  TString _defname(defname);
1465  if (_defname == "") _defname = GetFrameName();
1466  // get list of all parents of new default
1467  TList parents;
1468  TString ff = newdef;
1469  KVKinematicalFrame* f;
1470  do {
1471  f = get_parent_frame(ff);
1472  if (f) {
1473  parents.Add(f);
1474  ff = f->GetName();
1475  }
1476  } while (f);
1477  // modify tree structure
1478  TIter it(&parents);
1479  KVKinematicalFrame* newdframe = get_frame(newdef);
1480  KVKinematicalFrame* save_newdef = newdframe;
1481  KVFrameTransform trans, next_trans = newdframe->GetTransform();
1482  while ((f = (KVKinematicalFrame*)it())) {
1483  f->GetParticle()->GetListOfFrames()->Remove(newdframe);
1484  trans = next_trans;
1485  next_trans = f->GetTransform();
1486  newdframe->GetParticle()->GetListOfFrames()->Add(f);
1487  f->SetTransform(trans.Inverse());
1488  newdframe = f;
1489  }
1490  GetListOfFrames()->Remove(newdframe);
1491  // copy current default kinematics particle momentum/energy
1492  TLorentzVector old_def_p(*this);
1493  // set momentum/energy of new default kinematics
1494  Set4Mom(*(save_newdef->GetParticle()));
1495  save_newdef->GetParticle()->Set4Mom(old_def_p);
1496  save_newdef->SetTransform(next_trans.Inverse());
1497  save_newdef->SetName(_defname);
1498  newdframe->GetParticle()->GetListOfFrames()->Add(save_newdef);
1499  // copy frame list from new default frame
1500  TList frame_list;
1501  frame_list.AddAll(save_newdef->GetParticle()->GetListOfFrames());
1502  save_newdef->GetParticle()->GetListOfFrames()->Clear("nodelete");
1503  save_newdef->GetParticle()->GetListOfFrames()->AddAll(&fBoosted);
1504  fBoosted.Clear("nodelete");
1505  fBoosted.AddAll(&frame_list);
1506  SetFrameName(newdef);
1507 }

1571 void KVParticle::SetFrame(const Char_t* frame, const KVFrameTransform& ft)
1572 {
1573  //Define a Lorentz-boosted and/or rotated frame in which to calculate this particle's momentum and energy.
1574  //
1575  //The new frame will have the name given in the string "frame", which can then be used to
1576  //access the kinematics of the particle in different frames using GetFrame() (frame names are case-insensitive).
1577  //Calling this method with the name of an existing frame will update the kinematics of the particle
1578  //in that frame using the given transform (note that kinematics in all frames defined as 'subframes' of
1579  //this frame will also be updated).
1580  //
1581  //USING BOOSTS
1582  //The boost velocity vector is that of the boosted frame with respect to the original frame of the particles in the event.
1583  //The velocity vector can be given either in cm/ns units (default) or in units of 'c' (beta=kTRUE).
1584  //
1585  //E.g. to define a frame moving at 0.1c in the +ve z-direction with respect to the original
1586  //event frame:
1587  //
1588  // (...supposing a valid pointer KVParticle* my_part...)
1589  // TVector3 vframe(0,0,0.1);
1590  // my_part->SetFrame("my_frame", KVFrameTransform(vframe, kTRUE));
1591  //
1592  //or with velocity in cm/ns units (default):
1593  // TVector3 vframe(0,0,3);
1594  // my_part->SetFrame("my_frame", KVFrameTransform(vframe));
1595  // OR my_part->SetFrame("my_frame", vframe);
1596  //
1597  //USING ROTATIONS
1598  //According to the conventions adopted for the TRotation and TLorentzRotation classes,
1599  //we actually use the inverse of the TLorentzRotation to make the transformation,
1600  //to get the coordinates corresponding to a rotated coordinate system, not the coordinates
1601  //of a rotated vector in the same coordinate system
1602  //=> you do not need to invert the transformation matrix
1603  //
1604  //E.g. if you want to define a new frame whose coordinate axes are rotated with respect
1605  //to the original axes, you can set up a TRotation like so:
1606  //
1607  // TRotation rot;
1608  // TVector3 newX, newY, newZ; // the new coordinate axes
1609  // rot.RotateAxes(newX, newY, newZ);
1610  //
1611  //If you are using one of the two global variables which calculate the event tensor
1612  //(KVTensP and KVTensPCM) you can obtain the transformation to the tensor frame
1613  //using:
1614  //
1615  // TRotation rot;
1616  // KVTensP* tens_gv;// pointer to tensor global variable
1617  // tens_gv->GetTensor()->GetRotation(rot);// see KVTenseur3::GetRotation
1618  //
1619  //Then the new frame can be defined by
1620  // my_part->SetFrame("my_frame", KVFrameTransform(rot));
1621  // OR my_part->SetFrame("my_frame", rot);
1622  //
1623  //USING COMBINED BOOST AND ROTATION
1624  //You can define a frame using both a boost and a rotation like so:
1625  // my_part->SetFrame("my_frame", KVFrameTransform(vframe,rot,kTRUE));
1626  //
1627  //ACCESSING KINEMATICS IN NEW FRAMES
1628  //In order to access the kinematics of the particle in the new frame:
1629  //
1630  // my_part->GetFrame("my_frame")->GetTransverseEnergy();// transverse energy in "my_frame"
1631 
1632  if (!strcmp(frame, "")) return;
1633 
1634  KVKinematicalFrame* tmp = get_frame(frame);
1635  if (!tmp) {
1636  //if this frame has not already been defined, create a new one
1637  tmp = new KVKinematicalFrame(frame, this, ft);
1638  fBoosted.Add(tmp);
1639  } else
1640  tmp->ApplyTransform(this, ft);
1641 }

1683 KVParticle const* KVParticle::GetFrame(const Char_t* frame, Bool_t warn_and_return_null_if_unknown)
1684 {
1685  // Return the momentum of the particle in the Lorentz-boosted frame corresponding to the name
1686  // "frame" given as argument (see SetFrame() for definition of different frames).
1687  // If the default frame name has been set (see KVEvent::SetFrameName) and 'frame' is the
1688  // name of this default frame (KVParticle::fFrameName), we return the address of the particle
1689  // itself.
1690  //
1691  // This frame may have been defined by a direct transformation of the original kinematics of the
1692  // particle (using SetFrame(newframe,...)) or by a transformation of the kinematics in another
1693  // user-defined frame (using SetFrame(newframe,oldframe,...)).
1694  //
1695  // Note that frames are not "dynamic": if any changes are made to the original particle's kinematics
1696  // after definition of a frame, if you want these changes to affect also the other frames you
1697  // need to update them by hand by calling KVParticle::UpdateAllFrames().
1698  //
1699  // Frame names are case insensitive: "CM" or "cm" or "Cm" are all good...
1700  //
1701  // By default, if no frame with the given name is found, we return nullptr and print a warning.
1702  // If `warn_and_return_null_if_unknown=kFALSE`, we return the address of the particle itself,
1703  // i.e. the original/default kinematics.
1704  // [Note that this is an inversion of the previous default behaviour]
1705  //
1706  // Note that the properties of the particle returned by this method can not be modified:
1707  // this is deliberate, as any modifications e.g. to kinematics will have no effect
1708  // in any other frames.
1709  //
1710  // The returned pointer corresponds to a "pseudoparticle" in the desired frame,
1711  // therefore you can use any KVParticle method in order to access the kinematics of the
1712  // particle in the boosted frame, e.g.
1713  //
1714  //~~~~~~~~~~~~~~~~~~~~
1715  // (...supposing a valid pointer KVParticle* my_part...)
1716  // my_part->GetFrame("cm_frame")->GetVpar();// //el velocity in "cm_frame"
1717  // my_part->GetFrame("QP_frame")->GetTheta();// polar angle in "QP_frame"
1718  // etc. etc.
1719  //~~~~~~~~~~~~~~~~~~~~
1720  //
1721 
1722  if (!fFrameName.CompareTo(frame, TString::kIgnoreCase)) return (KVParticle const*)this;
1723  KVKinematicalFrame* f = get_frame(frame);
1724  return f ? (KVParticle const*)f->GetParticle() :
1725  (warn_and_return_null_if_unknown ?
1726  Warning("GetFrame(const Char_t*)", "No frame \"%s\" defined for particle. 0x0 returned.",
1727 #ifndef WITH_CPP11
1728  frame), (KVParticle*)nullptr
1729 #else
1730  frame), nullptr
1731 #endif
1732  : this);
1733 }
