21 <h2>KVLockfile</h2>
22 <h4>Interface to (Linux) system lockfile command</h4>
23 <p>lockfile can be used to create one or more semaphore files. If lockfile can't create all the specified files (in the
24  specified order), it waits sleeptime (defaults to 8) seconds and retries the last file that didn't succeed. You can
25  specify the number of retries to do until failure is returned. If the number of retries is -1 (default)
26  lockfile will retry forever.</p>
27 
28 <p> If the number of retries expires before all files have been created, lockfile returns failure and removes all the files
29  it created up till that point.</p>
30 
31 <p>If you specify a locktimeout then a lockfile will be removed by force after locktimeout seconds have passed since the
32  lockfile was last modified/created (most likely by some other program that unexpectedly died a long time ago, and hence
33  could not clean up any leftover lockfiles). Lockfile is clock skew immune. After a lockfile has been removed by
34  force, a suspension of suspend seconds (defaults to 16) is taken into account, in order to prevent the inadvertent
35  immediate removal of any newly created lockfile by another program (compare SUSPEND in procmail(1)).</p>
36 
37 <p>To use a lockfile, create a KVLockfile object either using the constructor without arguments
38 or by giving the name of the file to lock:</p>
39 <pre>
40 KVLockfile lox;
41 KVLockfile lox2("important.dat");
42 </pre>
43 <p>Then set the values you want for the number of retries, the sleeptime etc. Default values are as for
44 Linux 'lockfile':</p>
45 <pre>
46 sleeptime = 8 sec.
47 retries = -1 (keep trying for ever)
48 suspend = 16 sec. (in case of timeout)
49 </pre>
50 <p>There is no timeout by default. If you want one, use SetTimeout(timeout) with timeout>0 seconds.</p>
51 <p>To lock the file:</p>
52 <pre>
53 lox.Lock("important.dat"); // filename not given to ctor
54 lox2.Lock(); // filename given to ctor
55 </pre>
56 <p>To unlock the file:</p>
57 <pre>
58 lox.Release();
59 lox2.Release();
60 </pre>
61 <p>Note that Release() is called automatically in the dtor in case e.g. the KVLockfile goes out of scope.</p>

255 Bool_t KVLockfile::Lock(const Char_t* filename)
256 {
257  if (!have_exec) {
258  locked = kTRUE;
259  return kTRUE;
260  }
261  if (locked) {
262  cout << "<Error in KVLockfile::Lock: file " << fFile.Data() << " is already locked. Release it first>" << endl;
263  return kFALSE;
264  }
265  if (strcmp(filename, "")) fFile = filename;
266  writecmd();
267  if (!testlock()) {
268  //cout << "<Info in KVLockfile::Lock : Locked " << fFile.Data() << ">" << endl;
269  locked = kTRUE;
270  return kTRUE;
271  } else {
272  cout << "<Error in KVLockfile::Lock: can't get a lock for file " << fFile.Data() << ">" << endl;
273  locked = kFALSE;
274  return kFALSE;
275  }
276 }

284 Bool_t KVLockfile::Release()
285 {
286  if (!have_exec) {
287  locked = kFALSE;
288  return kTRUE;
289  }
290  if (!locked) {
