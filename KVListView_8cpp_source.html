23 <h3>Example of use</h3>
24 KVListView takes a list of objects (TObject or derived class) and presents them
25 as a list of items with different data for each object presented in separate columns.<br>
26 The user specifies the common base-class of the objects to display, the number of columns,
27 and the data to be presented in each column.
28 <br>
29 The example in the figure shown above (displaying a list of <a href="KVDBRun.html">KVDBRun</a>
30 objects) was generated with the following code:
31 <br>
32 <br>
33 <pre>
34  lvRuns = new KVListView(<a href="KVDBRun.html">KVDBRun</a>::Class(), cfRuns, 500, 250); // 'cfRuns' is pointer to parent frame
35 
36  lvRuns->SetDataColumns(4); //set number of columns to 4
37 
38  lvRuns->SetDataColumn(0, "Run", "GetNumber"); //by default, the data presented in a column with title "Toto" will be retrieved
39  lvRuns->SetDataColumn(1, "Events", "", kTextRight); //from each object by calling the "GetToto" method of the class given to the KVListView ctor.
40  lvRuns->SetDataColumn(2, "Date", "GetDatime"); //in case the "getter" method has a non-standard name, it can be given explicitly.
41 
42  lvRuns->GetDataColumn(2)-><a href="KVLVColumnData.html#KVLVColumnData:SetIsDateTime">SetIsDateTime()</a>; //some special treatment is afforded to date/time data. see method doc.
43  lvRuns->SetDataColumn(3, "Comments", "", kTextLeft);// specify text alignment for data
44 
45  lvRuns->ActivateSortButtons();//when clicking a column header, the list is sorted according to that column's data.
46  ... //clicking a second time the same column sorts objects in the opposite sense.
47 </pre>
48 <br>
49 To display objects, put them in a TList and call
50 <br>
51 <br>
52 <pre>
53  lvRuns->Display( pointer_to_TList );
54 </pre>
55 <br>
56 <h3>Object selection</h3>
57 Items in the list can be selected using either:
58  <ul>
59  <li>single left click (single object selection),</li>
60  <li>CTRL+left-click (selection of multiple non-neighbouring objects),</li>
61  <li>or SHIFT+left-click (selection of multiple & neighbouring objects)</li>
62  <li>CTRL+a (select all objects in list)</li>
63  </ul>
64 (By default, multiple selection of objects is allowed. Call AllowMultipleSelection(kFALSE) to disable).
65 <br>
66 <br>
67 When the selection changes, the KVListView emits the
68 <pre>
69  SelectionChanged()
70 </pre> signal.<br>
71 The list of the currently selected objects can be retrieved with the method
72 <pre>
73  GetSelectedObjects()
74 </pre>
75 which returns a TList of the currently selected objects (delete list after use), or
76 <pre>
77  GetPickOrderedSelectedObjects()
78 </pre>
79 which returns a TList of the currently selected objects IN THE ORDER THEY WERE CLICKED
80 - this list must NOT be deleted by the user.
81 <br>
82 <br>
83 <h3>Double-click action</h3>
84 By default, double-left-clicking an object will execute the
85 <pre>
86  Browse()
87 </pre>
88 method of the object. This method is defined for all TObjects,
89 it can be overridden in derived classes in order to do something interesting and/or useful.<br>
90 Alternatively, using method
91 <pre>
92  SetDoubleClickAction(const char* receiver_class, void* receiver, const char* slot)
93 </pre>
94 you can define a different behaviour when objects are double-clicked using signals & slots.<br>
95 If you want to deactivate the double-click actions, call
96 <pre>
97  AllowDoubleClick(kFALSE)
98 </pre>
99 <br>
100 <h3>Context menus</h3>
101 By default, context menus are active, which means that right-clicking an object
102 opens its context menu. Call method
103 <pre>
104  AllowContextMenu(kFALSE)
105 </pre>
106 to deactivate all context menus, or
107 <pre>
108  AddContextMenuClassException(TClass*)
109 </pre>
110 to activate/deactivate context menus for specific classes, depending on whether
111 context menus are globally activated or not.
112 <br>
113 See also 'Wrapper classes' below.
114 <br>
115 <h3>Icons</h3>
116 An icon is shown next to each item in the list. This icon is defined (for most ROOT objects
117 e.g. histograms of different types) in the $ROOTSYS/etc/root.mimes file. Other icons can be
118 added in .kvrootrc as in the following example:
119 <pre>
120  +KaliVeda.GUI.MimeTypes: TCutG
121  KaliVeda.GUI.MimeTypes.TCutG.Icon: cut_t.xpm
122 </pre>
123 which adds an icon (pair of scissors) for the TCutG class (not defined in root.mimes).
124 <br>
125 See also 'Wrapper classes' below.
126 <br>
127 <h3>Wrapper classes</h3>
128 For convenience, the objects placed in the list may be of a class which provides
129 a 'wrapper' around the actual object of interest.
130 <br>
131 In this case, it is possible (if the 'wrapper'
132 inherits from <a href="KVBase.html">KVBase</a>) to have the icon and the context menu
133 of each object reflect the object of interest, rather than the wrapper.
134 <br>
135 To do so, call method
136 <pre>
137  SetUseObjLabelAsRealClass(kTRUE)
138 </pre>
139 and have the wrapper class' <code>GetLabel()</code> and <code>GetObject()</code> methods return
140 the class name and address of the wrapped object.

306 Bool_t KVListView::ProcessMessage(Long_t msg, Long_t parm1, Long_t parm2)
307 {
308  // Handle messages generated by the list view container
309  switch (GET_SUBMSG(msg)) {
310  case kCT_SELCHANGED:
311  // only emit signal if number selected has really changed
312  if ((int)parm2 != nselected) {
313  nselected = (int)parm2;
314  SelectionChanged();
315  }
316  return kTRUE;
317  break;
318  default:
319  break;
320  }
321  return TGCanvas::ProcessMessage(msg, parm1, parm2);
322 }

404 void KVListView::SetDefaultColumnWidth(TGVFileSplitter* splitter)
405 {
406  // Set default column width of the columns headers.
407  // Limit minimum size of a column to total width / number of columns
408  // If only one column it will span the whole viewport
409 
410  TGLVContainer* container = (TGLVContainer*) fVport->GetContainer();
411 
412  if (!container) {
413  Error("SetDefaultColumnWidth", "no listview container set yet");
414  return;
415  }
416  container->ClearViewPort();
417  UInt_t minWidth = container->GetPageDimension().fWidth / (fNColumns - 1);
418 
419  for (int i = 0; i < fNColumns; ++i) {
420  if (fSplitHeader[i] == splitter) {
421  TString dt = fColHeader[i]->GetString();
422  UInt_t bsize = gVirtualX->TextWidth(fColHeader[i]->GetFontStruct(),
423  dt.Data(), dt.Length());
424  UInt_t w = TMath::Max(fColHeader[i]->GetDefaultWidth(), bsize + 20);
425  if (i == 0) {
426  //w = TMath::Max(fMaxSize.fWidth + 10, w);
427  w = TMath::Max(w, minWidth);
428  }
429  if (i > 0) {
430  w = TMath::Max(container->GetMaxSubnameWidth(i) + 40, (Int_t)w);
431  //printf("w=%ud\n",w);
432  //w = TMath::Min(w, fMaxColumnSize);
433  w = TMath::Max(w, minWidth);
434  }
435  fColHeader[i]->Resize(w, fColHeader[i]->GetHeight());
436  Layout();
437  }
438  }
439 }

494 void KVListView::AddContextMenuClassException(TClass* cl)
495 {
496  // The global context menu status (allowed or not allowed) is set by AllowContextMenu().
497  // If required, this can be overridden for specific classes by calling this
498  // method for each required class.
499  // In this case, any objects in the list of precisely this class (not derived classes)
500  // will have the opposite behaviour to that defined by AllowContextMenu(),
501  // i.e. if context menus are globally disabled, this method defines the classes for
502  // which a context menu is authorised, and vice-versa.
503 
504  ((KVLVContainer*)GetContainer())->AddContextMenuClassException(cl);
505 }
